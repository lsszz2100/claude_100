# 119. 비개발자 업무 자동화 예시

저자: AI_Innovation_Studio

Claude Code는 개발자만의 도구가 아닙니다. 파일을 읽고 정리하고, 문서를 만들고, 반복 절차를 자동화하는 작업에도 사용할 수 있습니다. 단, 비개발자 업무에서는 dry run과 승인 절차가 특히 중요합니다.

## 자동화 후보

| 업무 | 자동화 방식 |
| --- | --- |
| 회의록 정리 | 액션 아이템, 담당자, 기한 추출 |
| FAQ 작성 | 고객 문의를 주제별로 묶어 초안 생성 |
| 파일명 정리 | 규칙 제안 후 승인된 파일만 변경 |
| 보고서 초안 | 자료 요약과 표준 형식 적용 |
| 이메일 분류 | 유형, 우선순위, 답변 초안 생성 |
| 문서 점검 | 누락 항목, 오래된 정보, 중복 찾기 |

## 안전한 요청 패턴

```text
아래 폴더의 파일을 정리하고 싶다.

규칙:
- 먼저 현재 파일 목록을 분석한다.
- 변경 계획을 표로 제안한다.
- 실제 이름 변경은 내가 승인하기 전까지 하지 않는다.
- 삭제는 제안만 하고 실행하지 않는다.

출력:
1. 현재 상태 요약
2. 추천 분류 기준
3. 변경 전/후 파일명 표
4. 위험한 항목
```

## 좋은 자동화의 조건

- 반복 빈도가 높다.
- 판단 기준을 문장으로 설명할 수 있다.
- 실패해도 되돌릴 수 있다.
- 민감 정보가 적거나 마스킹 가능하다.
- 실행 전 미리보기로 검토할 수 있다.

## 업무별 첫 실습

| 업무 | 첫 실습 |
| --- | --- |
| 회의록 | 1개 파일에서 결정 사항과 할 일을 표로 추출 |
| FAQ | 문의 10개를 주제별로 묶고 답변 초안 작성 |
| 파일 정리 | 파일명을 바꾸지 말고 변경 계획만 표로 작성 |
| 보고서 | 기존 보고서 1개를 기준으로 새 목차 제안 |
| 이메일 | 답장을 보내지 말고 우선순위와 답변 초안만 작성 |

처음부터 폴더 전체를 맡기지 마세요. 샘플 5개에서 규칙이 맞는지 확인한 뒤 범위를 늘리는 편이 안전합니다.

## 민감 정보 기준

비개발자 업무 문서에는 고객명, 계약 조건, 인사 정보, 내부 계정이 섞이기 쉽습니다.

| 데이터 | 기본 처리 |
| --- | --- |
| 고객명과 연락처 | 마스킹 후 요약 |
| 계약 금액과 조건 | 승인 전 외부 전송 금지 |
| 인사 평가 | 자동화 대상에서 제외 |
| 계정과 비밀번호 | 입력 금지 |
| 회의록 | 참석자와 외부 공유 범위 확인 |

## 승인 단계 예시

```text
방금 제안한 변경 계획 중 위험한 항목을 제외하고,
실행해도 되는 항목과 보류할 항목을 나눠줘.
실제 파일 수정은 아직 하지 마.
```

## 체크리스트

- [ ] dry run 결과를 먼저 봤다.
- [ ] 삭제보다 분류와 이름 변경부터 시작했다.
- [ ] 원본 백업이나 되돌릴 방법이 있다.
- [ ] 민감한 문서는 제외했다.
- [ ] 처음에는 샘플 파일로만 규칙을 검증했다.
- [ ] 실행 전 변경 전/후 표를 검토했다.
- [ ] 외부 전송이 필요한 작업은 별도 승인 단계로 분리했다.

## 다음 단계

비개발자 업무 자동화는 작은 폴더 하나, 문서 종류 하나, 반복 보고서 하나에서 시작하세요. 기준이 안정되면 Skill이나 스크립트로 패키징할 수 있습니다.
