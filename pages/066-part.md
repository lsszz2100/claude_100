# Part 7. Skills

이 장은 고급 단계 독자를 위한 흐름입니다. 각 페이지는 하나의 주제를 다루며, 개념 설명과 실습 프롬프트를 함께 제공합니다.

## 처음 읽는 분께

이 파트는 용어를 외우는 장이 아니라 실제 작업에서 판단 기준을 세우는 장입니다. 각 항목을 읽을 때는 “언제 쓰는가”, “무엇을 조심해야 하는가”, “어떻게 검증하는가”를 함께 확인하세요.

초보자는 본문을 읽은 뒤 바로 실습 프롬프트를 실행해 보는 것이 좋습니다. 실행 결과가 예상과 다르면 실패로 보지 말고, 어떤 맥락이 부족했는지 기록하세요. 그 기록이 다음 요청을 더 정확하게 만드는 재료가 됩니다.

## 이 장에서 다루는 100선 항목

- 066. Skills 개념: Skill은 반복 가능한 능력을 파일, 스크립트, 템플릿으로 패키징한 단위다.
- 067. SKILL.md 구조: 이름, 설명, 호출 조건, 절차, 참조 파일을 명확히 작성한다.
- 068. slash commands와 skills 차이: legacy command와 현재 권장되는 skill 방식의 차이를 이해한다.
- 069. 코드 리뷰 Skill 만들기: 리뷰 기준, 심각도, 출력 형식, 테스트 공백을 Skill에 담는다.
- 070. 테스트 Skill 만들기: 테스트 전략, 실행 명령 탐색, 실패 해석 방식을 재사용 가능하게 만든다.
- 071. 문서화 Skill 만들기: API 문서, 사용자 가이드, 변경 로그를 일관된 구조로 생성한다.
- 072. 배포 체크리스트 Skill 만들기: 배포 전 확인 항목과 롤백 절차를 자동으로 점검한다.
- 073. Skill의 progressive disclosure: 처음부터 모든 문서를 로드하지 않고 필요한 자료만 열게 한다.
- 074. 쓸모없는 Skill 제거 기준: 모델 기본 능력을 방해하거나 컨텍스트만 낭비하는 Skill을 제거한다.

## 학습 방식

각 항목을 읽은 뒤 바로 Claude 또는 Claude Code에 실습 프롬프트를 입력해 보세요. 읽기만 하면 기능 목록으로 남고, 직접 실행하면 작업 습관으로 바뀝니다.
