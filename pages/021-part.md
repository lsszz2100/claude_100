# Part 03. Claude Code 기본 명령

이 장은 초급 단계 독자를 위한 흐름입니다. 각 페이지는 하나의 주제를 다루며, 개념 설명과 실습 프롬프트를 함께 제공합니다.

## 처음 읽는 분께

이 파트는 용어를 외우는 장이 아니라 실제 작업에서 판단 기준을 세우는 장입니다. 각 항목을 읽을 때는 “언제 쓰는가”, “무엇을 조심해야 하는가”, “어떻게 검증하는가”를 함께 확인하세요.

초보자는 본문을 읽은 뒤 바로 실습 프롬프트를 실행해 보는 것이 좋습니다. 실행 결과가 예상과 다르면 실패로 보지 말고, 어떤 맥락이 부족했는지 기록하세요. 그 기록이 다음 요청을 더 정확하게 만드는 재료가 됩니다.

## 이 장에서 다루는 100선 항목

- 021. /help, /clear, /compact, /cost: 기본 slash command로 세션, 컨텍스트, 비용을 관리한다.
- 022. 파일 읽기와 코드베이스 요약: Claude Code가 필요한 파일을 찾아 읽고 요약하게 만드는 법을 익힌다.
- 023. 작은 버그 고치기: 작은 범위의 문제를 재현, 원인 분석, 수정, 검증 순서로 처리한다.
- 024. 테스트 실행 요청하기: 수정 후 테스트를 실행하고 실패를 읽어 다시 고치는 반복을 배운다.
- 025. Git 변경사항 확인하기: 수정된 파일과 diff를 확인한 뒤 다음 행동을 결정한다.
- 026. 커밋 메시지 작성시키기: 변경 의도와 범위를 반영한 커밋 메시지 작성법을 익힌다.
- 027. README 요약시키기: 문서에서 설치, 실행, 테스트, 배포 정보를 추출한다.
- 028. 비개발자를 위한 Claude Code 활용: 코딩 지식이 적어도 파일 정리, 문서 분석, 자동화 도구 제작에 활용할 수 있음을 이해한다.
- 029. 학습자용 Claude 사용법: 정답을 복사하는 대신 설명, 힌트, 연습문제를 요청하는 학습법을 익힌다.
- 030. 초급자가 피해야 할 실수: 무작정 전체 수정, 테스트 생략, 권한 무비판 승인, 비밀정보 노출을 피한다.

## 학습 방식

각 항목을 읽은 뒤 바로 Claude 또는 Claude Code에 실습 프롬프트를 입력해 보세요. 읽기만 하면 기능 목록으로 남고, 직접 실행하면 작업 습관으로 바뀝니다.
