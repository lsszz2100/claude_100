# 060. 원격/모바일 작업 흐름

난이도: 중급  
기준일: 2026년 05월 03일

## 핵심 개념

Claude Code 작업은 반드시 노트북 앞에서만 시작할 필요는 없습니다. Claude Code Web과 모바일 앱을 함께 쓰면 이동 중에 작업을 맡기고, 진행 상황을 확인하고, 필요한 경우 방향을 조정할 수 있습니다.

다만 모바일은 깊은 코드 리뷰보다 작업 지시, 진행 확인, 중단 판단에 더 적합합니다.

## 모바일에 맞는 작업

| 적합 | 부적합 |
| --- | --- |
| 명확한 문서 수정 요청 | 복잡한 UI 픽셀 조정 |
| 작은 테스트 추가 | 비밀정보가 필요한 배포 작업 |
| 버그 재현 로그 전달 | 긴 diff 세부 리뷰 |
| 작업 진행 모니터링 | 대규모 리팩터링 즉석 승인 |
| PR 설명 초안 요청 | 운영 DB migration 승인 |

모바일에서는 승인 실수를 줄이기 위해 destructive 작업을 피하는 것이 좋습니다.

## 모바일 작업 지시 템플릿

```text
이 작업은 모바일에서 맡기는 비동기 작업이다.

목표:
[명확한 목표]

범위:
[파일/폴더]

금지:
- 파일 삭제
- migration 실행
- 새 의존성 추가
- 배포 명령 실행

완료 기준:
- [테스트 또는 문서 검증]

완료 후:
- 변경 파일
- 실행한 명령
- 실패한 검증
- 다음에 내가 확인할 사항
을 요약해줘.
```

## 원격에서 로컬로 넘기기

원격 작업 후 로컬에서 이어받을 때는 인수인계가 필요합니다.

```text
원격 세션에서 진행한 작업을 로컬에서 이어받기 위한 요약을 만들어줘.

포함할 것:
1. 작업 목표
2. 변경 파일
3. 주요 결정
4. 실행한 테스트
5. 실패 또는 미실행 검증
6. 남은 TODO
7. 로컬에서 가장 먼저 확인할 명령
```

## 중단 기준

모바일에서 진행 중인 작업이 다음 조건에 걸리면 멈추게 하세요.

- migration이 필요해졌다.
- 새 secret이나 credential이 필요하다.
- 테스트가 대량 실패한다.
- 변경 파일이 예상보다 많아졌다.
- 권한이나 보안 설계가 모호하다.
- 배포나 운영 데이터 접근이 필요하다.

프롬프트에 중단 조건을 넣을 수 있습니다.

```text
작업 중 위 조건 중 하나라도 발생하면 수정하지 말고 멈춘 뒤 나에게 질문해줘.
```

## 체크리스트

- [ ] 모바일에서는 작은 비동기 작업만 맡긴다.
- [ ] 삭제, 배포, migration, secret 작업은 금지한다.
- [ ] 중단 조건을 프롬프트에 넣는다.
- [ ] 로컬로 이어받을 요약을 받는다.
- [ ] 최종 병합 전 로컬에서 diff와 테스트를 확인한다.
