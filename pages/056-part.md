# Part 6. 작업 환경 확장

이 장은 중급 단계 독자를 위한 흐름입니다. 각 페이지는 하나의 주제를 다루며, 개념 설명과 실습 프롬프트를 함께 제공합니다.

## 처음 읽는 분께

이 파트는 용어를 외우는 장이 아니라 실제 작업에서 판단 기준을 세우는 장입니다. 각 항목을 읽을 때는 “언제 쓰는가”, “무엇을 조심해야 하는가”, “어떻게 검증하는가”를 함께 확인하세요.

초보자는 본문을 읽은 뒤 바로 실습 프롬프트를 실행해 보는 것이 좋습니다. 실행 결과가 예상과 다르면 실패로 보지 말고, 어떤 맥락이 부족했는지 기록하세요. 그 기록이 다음 요청을 더 정확하게 만드는 재료가 됩니다.

## 이 장에서 다루는 100선 항목

- 056. Claude Code와 VS Code: IDE diff, 선택 영역, 파일 탐색과 Claude Code의 역할 분담을 이해한다.
- 057. Claude Code와 JetBrains: JetBrains 기반 프로젝트에서 Claude Code를 보조 개발자로 쓰는 방식을 익힌다.
- 058. Claude Code Desktop 활용: 시각적 diff 검토, 멀티 세션, 예약 작업의 장점을 이해한다.
- 059. Claude Code Web 활용: 로컬 환경 없이 장기 작업과 병렬 작업을 시작하는 방식을 이해한다.
- 060. 원격/모바일 작업 흐름: 세션이 하나의 표면에 묶이지 않고 이동할 수 있다는 점을 활용한다.
- 061. 세션 resume 전략: 긴 작업에서 맥락을 유지하되 오래된 잡음을 정리하는 법을 배운다.
- 062. context window 관리: 컨텍스트는 무한하지 않으므로 요약, 제외, 파일화 전략이 필요하다.
- 063. 70%, 85%, 90% 컨텍스트 압력 대응: 컨텍스트 압력이 높아질수록 정밀도가 낮아질 수 있음을 이해한다.
- 064. 비용과 모델 선택: 일상 작업과 복잡한 설계 작업에 다른 모델/노력을 배분한다.
- 065. 중급자가 만드는 첫 자동화 시스템: 문서 요약, 이슈 정리, 테스트 실행 같은 작고 유용한 자동화를 설계한다.

## 학습 방식

각 항목을 읽은 뒤 바로 Claude 또는 Claude Code에 실습 프롬프트를 입력해 보세요. 읽기만 하면 기능 목록으로 남고, 직접 실행하면 작업 습관으로 바뀝니다.
