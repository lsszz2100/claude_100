# 002. Claude와 ChatGPT의 차이

난이도: 초급  
기준일: 2026년 05월 03일

## 핵심 개념

Claude와 ChatGPT는 모두 대규모 언어 모델을 기반으로 한 AI 도구입니다. 둘 다 질문에 답하고, 글을 쓰고, 코드를 설명하고, 아이디어를 정리할 수 있습니다. 그러나 실제 사용 경험은 모델의 성능만으로 결정되지 않습니다. 어떤 인터페이스에서 쓰는지, 어떤 파일과 도구에 접근할 수 있는지, 사용자가 어떤 방식으로 작업을 맡기는지가 더 큰 차이를 만듭니다.

이 책은 특정 제품을 단순 비교하는 데 목적이 있지 않습니다. 중요한 것은 Claude를 사용할 때 무엇을 기대해야 하고, Claude Code가 일반 채팅형 AI와 어떻게 다른지를 이해하는 것입니다.

## 차이를 만드는 요소

첫째, 작업 공간입니다. 일반 챗봇은 대화창 안에서 답변을 생성합니다. 반면 Claude Code는 프로젝트 폴더 안에서 작동하며 파일을 읽고 수정할 수 있습니다.

둘째, 도구 사용입니다. Claude Code는 터미널 명령, Git, 테스트, 린트, 빌드, MCP 서버, IDE 통합과 연결될 수 있습니다. 이때 Claude는 단순히 “무엇을 해야 하는지” 말하는 것이 아니라 “실제로 해보고 결과를 읽는” 방식으로 일합니다.

셋째, 지속성입니다. 웹 대화는 대화 맥락에 의존하지만, Claude Code 프로젝트는 `CLAUDE.md`, `.claude/skills`, `.mcp.json`, hooks 같은 파일 기반 설정을 통해 반복 가능한 작업 환경을 만들 수 있습니다.

넷째, 검증 방식입니다. 채팅형 AI의 답변은 사용자가 직접 확인해야 합니다. Claude Code도 검증이 필요하지만, 테스트 실행, 빌드, diff 확인, Git 상태 확인 같은 개발 검증 루프에 더 자연스럽게 들어갑니다.

## 비교표

| 구분 | 일반 채팅형 AI | Claude Code |
| --- | --- | --- |
| 기본 공간 | 대화창 | 프로젝트 폴더 |
| 주요 결과 | 텍스트 답변 | 파일 수정, 명령 실행, 코드 검증 |
| 맥락 관리 | 대화 맥락 중심 | 파일, 메모리, 설정 중심 |
| 검증 | 사용자가 수동 확인 | 테스트/빌드/Git과 연결 가능 |
| 적합한 작업 | 글쓰기, 요약, 아이디어 | 개발, 리팩터링, 자동화, 코드 리뷰 |

## 왜 중요한가

도구의 차이를 모르고 사용하면 기대가 어긋납니다. 예를 들어 웹 채팅창에서 “내 프로젝트를 고쳐줘”라고 하면 Claude는 실제 파일을 볼 수 없으므로 일반적인 조언만 할 수 있습니다. 반대로 Claude Code에서 같은 요청을 하면 프로젝트 파일을 읽고 실제 수정까지 진행할 수 있습니다. 따라서 사용자는 “이 작업은 대화로 충분한가, 실제 파일 접근이 필요한가?”를 먼저 판단해야 합니다.

## 실습

같은 목표를 두 가지 방식으로 써 보세요.

```text
내 로그인 기능을 개선해줘.
```

이 요청은 너무 넓습니다. Claude Code용으로 바꾸면 다음처럼 됩니다.

```text
이 프로젝트의 로그인 기능을 조사해줘.
먼저 관련 파일을 찾아 구조를 설명하고, 보안과 사용자 경험 관점의 개선점을 제안해줘.
아직 파일은 수정하지 말고 계획만 작성해줘.
```

## Claude 또는 Claude Code에 입력할 프롬프트

```text
Claude Web에서 할 일과 Claude Code에서 할 일을 구분하고 싶다.
다음 작업 목록을 보고 Web, Code, Desktop, IDE 중 어디가 적합한지 표로 분류해줘.

작업:
- 블로그 글 초안 작성
- React 컴포넌트 버그 수정
- PR 리뷰
- 회의록 요약
- 테스트 실패 원인 분석
- 배포 전 체크리스트 작성
- 프로젝트 구조 설명
```

## 체크리스트

- [ ] 대화형 AI와 코드 에이전트의 차이를 설명할 수 있다.
- [ ] 파일 접근이 필요한 작업과 필요 없는 작업을 구분할 수 있다.
- [ ] Claude Code에서 바로 수정시키기 전에 계획을 요구할 수 있다.
- [ ] 도구 선택이 결과 품질에 영향을 준다는 점을 이해한다.
