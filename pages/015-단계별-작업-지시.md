# 015. 단계별 작업 지시

난이도: 초급  
기준일: 2026년 05월 03일
저자: AI_Innovation_Studio

## 핵심 개념

큰 작업을 한 문장으로 맡기면 Claude는 많은 결정을 스스로 해야 합니다. 작은 작업은 괜찮지만, 코드베이스 수정이나 자동화 설계처럼 영향 범위가 큰 작업에서는 위험합니다. 단계별 작업 지시는 Claude가 순서대로 생각하고 실행하게 만드는 방법입니다.

기본 순서는 다음과 같습니다.

1. 조사
2. 분석
3. 계획
4. 승인
5. 실행
6. 검증
7. 요약

## 나쁜 예

```text
검색 기능을 고쳐줘.
```

이 요청은 너무 짧습니다. Claude가 어디를 볼지, 어떤 증상인지, 테스트를 실행해야 하는지 알 수 없습니다.

## 좋은 예

```text
검색 기능이 빈 결과를 반환하는 문제가 있다.
바로 수정하지 말고 다음 순서로 진행해줘.

1. 검색 관련 파일을 찾아줘.
2. 현재 검색 흐름을 설명해줘.
3. 가능한 원인을 3개 이하로 정리해줘.
4. 가장 가능성 높은 원인과 수정 계획을 제안해줘.
5. 내가 승인하면 수정해줘.
6. 수정 후 테스트 또는 검증 방법을 실행해줘.
7. 변경 파일과 결과를 요약해줘.
```

이 요청은 Claude Code가 안전하게 움직이도록 합니다.

## 단계별 지시가 필요한 작업

- 버그 수정
- 리팩터링
- 데이터베이스 변경
- 인증/권한 수정
- 결제/보안 관련 작업
- 여러 파일을 건드리는 기능 개발
- 배포 자동화
- MCP나 Hooks 설정

## 단계별 지시가 덜 필요한 작업

- 오타 수정
- 짧은 설명 요청
- 간단한 비교표 작성
- 작은 코드 조각 설명

모든 요청을 복잡하게 만들 필요는 없습니다. 작업의 위험도가 높을수록 단계별 지시가 필요합니다.

## Claude Code용 기본 템플릿

```text
다음 작업을 단계별로 진행해줘.

작업:
[작업 설명]

진행 규칙:
1. 먼저 관련 파일을 찾아.
2. 현재 구조를 설명해.
3. 수정 계획을 제안해.
4. 내가 승인하기 전에는 파일을 수정하지 마.
5. 승인 후에는 최소 범위로 수정해.
6. 가능한 테스트를 실행해.
7. 변경 파일, 검증 결과, 남은 위험을 요약해.
```

## 실습

아래 요청을 단계별 지시로 바꿔 보세요.

```text
회원가입 오류를 고쳐줘.
```

힌트:

- 오류 메시지가 있는가?
- 재현 방법이 있는가?
- 관련 화면이나 API가 있는가?
- 테스트가 있는가?
- 바로 수정해도 되는가?

## 체크리스트

- [ ] 조사와 수정을 분리했다.
- [ ] 승인 전 파일 수정 금지를 명시했다.
- [ ] 검증 방법을 포함했다.
- [ ] 변경 후 요약을 요청했다.
- [ ] 작업 범위를 제한했다.

## 흔한 실수

- “단계별로 해줘”라고만 말하고 실제 단계를 주지 않는다.
- 계획과 실행을 한 번에 승인한다.
- 검증 단계를 빠뜨린다.
- 실패했을 때 되돌릴 방법을 고려하지 않는다.
