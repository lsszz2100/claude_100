# 095. 문서 작성자 에이전트

난이도: 고급  
기준일: 2026년 05월 03일
저자: AI_Innovation_Studio

## 핵심 개념

문서 작성자 에이전트는 코드와 설정을 근거로 README, API 문서, runbook, 릴리스 노트를 작성하거나 검토합니다. 핵심은 근거 없는 추정을 막고 독자별 문서 형식을 유지하는 것입니다.

문서 에이전트는 글을 예쁘게 쓰는 역할이 아니라, 현재 시스템과 문서의 차이를 줄이는 역할입니다.

## 설정 예시

````markdown
---
name: docs-writer
description: Create and review documentation from current code, configs, tests, and diffs. Use for README updates, API docs, runbooks, onboarding guides, and release notes.
tools: Read, Grep, Glob
---

You are a documentation specialist.

Rules:
- Identify the target audience first.
- Use current files as evidence.
- Mark unsupported claims as "확인 필요".
- Do not include secrets, tokens, or customer data.
- End with source files used.

Output:
- Audience:
- Purpose:
- Draft:
- Verification:
- Source files:
````

## 사용 예시

```text
문서 작성자 에이전트를 사용해 README의 실행 방법을 현재 코드 기준으로 검토해줘.

확인할 것:
1. 설치 명령
2. 개발 서버 실행 명령
3. 테스트 명령
4. 빌드 명령
5. 환경 변수 설명
6. 오래되었거나 확인 필요한 내용

아직 README를 수정하지 말고 검토 결과와 수정 초안만 작성해줘.
```

## 문서 유형별 초점

| 문서 | 초점 |
| --- | --- |
| README | 설치, 실행, 테스트, 프로젝트 개요 |
| API 문서 | 엔드포인트, 요청/응답, 오류, 인증 |
| Runbook | 장애 증상, 확인 명령, 복구 절차 |
| 릴리스 노트 | 사용자 영향, 변경, 주의사항 |
| 온보딩 문서 | 신규 팀원의 첫 실행 경로 |

## 근거 수준 표시

문서 작성자 에이전트는 문장을 근거 수준별로 구분해야 합니다.

| 수준 | 예시 | 처리 |
| --- | --- | --- |
| 확인됨 | `package.json`의 script, 실제 설정 파일 | 문서에 반영 |
| 추정 | 코드 구조상 예상되는 사용법 | “확인 필요” 표시 |
| 미확인 | 외부 서비스 설정, 배포 환경 | 사용자 확인 요청 |
| 금지 | secret, 고객 데이터, 내부 토큰 | 문서에 포함하지 않음 |

특히 README와 runbook은 오래 남는 문서입니다. 에이전트가 확신하지 못하는 내용을 사실처럼 쓰면 이후 작업자가 잘못된 절차를 반복하게 됩니다.

## 근거 표시

문서 작성자 에이전트는 사용한 파일을 끝에 남겨야 합니다.

```text
Source files:
- package.json
- README.md
- src/server.ts
- docs/api.md
```

## 실패 패턴

- 코드 확인 없이 일반적인 설치법을 쓴다.
- 독자가 개발자인지 운영자인지 구분하지 않는다.
- 환경 변수 예시에 실제 secret을 포함한다.
- 오래된 문서를 덮어쓰며 기존 중요한 경고를 삭제한다.
- 확인하지 못한 내용을 사실처럼 쓴다.

## 문서 변경 전 확인

문서 파일을 직접 수정하기 전에는 다음을 먼저 확인하게 하세요.

- 대상 독자가 개발자인가, 운영자인가, 사용자인가
- 기존 문서에서 유지해야 할 경고나 제약이 있는가
- 코드와 문서가 충돌하는 부분이 있는가
- 자동 생성 문서인지 사람이 수정하는 문서인지
- 외부 공개 문서인지 내부 문서인지

## 체크리스트

- [ ] 대상 독자가 분명하다.
- [ ] 현재 파일을 근거로 작성했다.
- [ ] 확인 필요 항목이 따로 표시된다.
- [ ] secret과 고객 데이터가 포함되지 않는다.
- [ ] 근거 수준이 확인됨, 추정, 미확인으로 구분된다.
- [ ] 기존 문서의 중요한 경고를 보존했다.
