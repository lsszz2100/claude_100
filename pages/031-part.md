# Part 4. 프로젝트 메모리

이 장은 중급 단계 독자를 위한 흐름입니다. 각 페이지는 하나의 주제를 다루며, 개념 설명과 실습 프롬프트를 함께 제공합니다.

## 이 장에서 다루는 100선 항목

- 031. CLAUDE.md의 역할: CLAUDE.md가 프로젝트의 장기 지침과 팀 규칙을 담는 파일임을 이해한다.
- 032. 프로젝트 메모리 설계: 자동으로 로드될 정보와 필요할 때만 읽을 정보를 분리한다.
- 033. 개인 메모리와 팀 메모리: 개인 선호와 팀 표준이 섞이면 협업이 어려워진다는 점을 이해한다.
- 034. 디렉터리별 CLAUDE.md: 특정 하위 폴더에만 적용할 규칙을 지역화한다.
- 035. .claudeignore로 컨텍스트 절약: 빌드 산출물, 의존성, 로그, 대용량 파일을 제외해 컨텍스트 낭비를 줄인다.
- 036. PRD.md 작성법: 제품 요구사항을 목표, 사용자, 범위, 비범위, 수용 기준으로 정리한다.
- 037. progress.md로 장기 작업 관리: 긴 세션과 여러 날 작업에서 진행 상태를 파일로 유지한다.
