# 120. 최종 프로젝트 10개

이 장은 책 전체를 실제 결과물로 연결하는 프로젝트 목록입니다. 한 번에 모두 만들 필요는 없습니다. 지금 가장 자주 반복하는 업무와 가장 작은 성공 기준을 기준으로 하나를 고르세요.

## 1. 프로젝트 온보딩 자동화

README, 설정 파일, 폴더 구조를 읽고 신규 팀원용 요약과 실행 체크리스트를 생성합니다.

완료 기준:
- 설치, 실행, 테스트 명령이 정리된다.
- 핵심 폴더와 파일의 역할이 설명된다.
- 신규 팀원이 따라 할 첫 작업이 포함된다.

## 2. PR 리뷰 자동화

diff를 읽고 버그, 보안, 테스트 공백 중심 리뷰를 작성하는 Skill과 subagent를 만듭니다.

완료 기준:
- 심각도별 findings 형식이 있다.
- 파일과 라인 근거가 포함된다.
- 칭찬보다 위험과 누락을 우선한다.

## 3. 실패한 CI 분석 도구

CI 로그에서 최초 실패 원인과 파생 오류를 구분하고 수정 후보를 제안합니다.

완료 기준:
- 최초 실패 지점을 찾는다.
- 관련 파일 후보를 제안한다.
- 재실행할 검증 명령을 남긴다.

## 4. README 검증 시스템

README 명령과 실제 package/build 설정이 일치하는지 검사합니다.

완료 기준:
- 문서 명령과 실제 스크립트를 비교한다.
- 오래된 설치 절차를 찾는다.
- 수정 제안을 PR 설명 형식으로 만든다.

## 5. RAG 기반 사내 문서 Q&A

문서 수집, 청킹, 검색, 출처 표시, 평가셋을 포함한 작은 RAG 시스템을 설계합니다.

완료 기준:
- 출처가 답변에 표시된다.
- 모르는 질문에 억지로 답하지 않는다.
- 검색 품질과 답변 품질을 따로 평가한다.

## 6. MCP 보안 감사

연결된 MCP 서버의 tools, 권한, 인증, 데이터 노출 위험을 감사표로 정리합니다.

완료 기준:
- 읽기/쓰기 권한이 구분된다.
- 토큰 저장 위치와 철회 절차가 확인된다.
- 과도한 권한을 줄이는 제안이 있다.

## 7. 배포 전 체크리스트 Skill

테스트, migration, 환경 변수, rollback, monitoring을 확인하는 배포 Skill을 만듭니다.

완료 기준:
- 배포 전 필수 확인 항목이 있다.
- 실패 시 중단 기준이 있다.
- rollback과 모니터링 확인이 포함된다.

## 8. 장기 작업 progress 시스템

`progress.md`, `/compact`, resume 전략을 묶어 여러 날 작업을 안정적으로 이어갑니다.

완료 기준:
- 완료, 진행 중, 남은 TODO가 분리된다.
- 테스트 결과와 실패한 명령이 기록된다.
- 다음 세션 첫 요청이 준비된다.

## 9. 비개발자 문서 자동화

회의록, FAQ, 보고서, 파일명 정리 작업을 dry run 중심으로 자동화합니다.

완료 기준:
- 실제 변경 전 미리보기 표가 있다.
- 삭제와 외부 전송은 승인 단계로 분리된다.
- 반복 가능한 템플릿이 남는다.

## 10. Claude Code 운영 표준

`CLAUDE.md`, Skills, Hooks, Subagents, MCP, CI를 포함한 팀 표준 템플릿을 만듭니다.

완료 기준:
- 팀 규칙과 개인 취향이 분리된다.
- 권한과 보안 기준이 문서화된다.
- 신규 프로젝트에 복사 가능한 최소 템플릿이 있다.

## 선택 기준

- 지금 반복해서 하고 있는가?
- 결과가 작게 검증 가능한가?
- 실패해도 되돌릴 수 있는가?
- 팀이나 개인에게 즉시 시간이 절약되는가?
