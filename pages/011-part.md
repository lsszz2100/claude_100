# Part 02. 프롬프트 기초

이 장은 초급 단계 독자를 위한 흐름입니다. 각 페이지는 하나의 주제를 다루며, 개념 설명과 실습 프롬프트를 함께 제공합니다.

## 처음 읽는 분께

이 파트는 용어를 외우는 장이 아니라 실제 작업에서 판단 기준을 세우는 장입니다. 각 항목을 읽을 때는 “언제 쓰는가”, “무엇을 조심해야 하는가”, “어떻게 검증하는가”를 함께 확인하세요.

초보자는 본문을 읽은 뒤 바로 실습 프롬프트를 실행해 보는 것이 좋습니다. 실행 결과가 예상과 다르면 실패로 보지 말고, 어떤 맥락이 부족했는지 기록하세요. 그 기록이 다음 요청을 더 정확하게 만드는 재료가 됩니다.

## 이 장에서 다루는 100선 항목

- 011. 좋은 질문과 나쁜 질문: 모호한 요청과 실행 가능한 요청의 차이를 배운다.
- 012. 프롬프트의 4요소: 역할, 목표, 맥락, 제약: 역할, 목표, 맥락, 제약을 조합하면 결과 품질이 안정된다는 원리를 익힌다.
- 013. 결과 형식 지정하기: 표, 체크리스트, JSON, 단계별 설명처럼 출력 형식을 직접 지정하는 법을 배운다.
- 014. 예시 기반 프롬프트: 원하는 결과 예시를 주면 Claude가 스타일과 구조를 더 안정적으로 맞춘다.
- 015. 단계별 작업 지시: 큰 요청을 탐색, 계획, 실행, 검증으로 나누는 법을 익힌다.
- 016. Claude에게 설명시키는 법: 학습용 설명, 코드 설명, 결정 이유 설명을 구분한다.
- 017. Claude에게 비교시키는 법: 도구, 라이브러리, 설계안을 비교할 때 평가 기준을 먼저 정해야 한다.
- 018. Claude에게 검토시키는 법: 검토는 칭찬보다 결함, 위험, 누락, 테스트 공백을 찾는 작업임을 이해한다.
- 019. Claude에게 계획시키는 법: 바로 구현하지 않고 먼저 계획을 만들게 하는 습관을 익힌다.
- 020. Plan Mode 기초: 복잡한 작업에서는 계획 승인 후 실행하는 방식이 안전하다는 점을 배운다.

## 학습 방식

각 항목을 읽은 뒤 바로 Claude 또는 Claude Code에 실습 프롬프트를 입력해 보세요. 읽기만 하면 기능 목록으로 남고, 직접 실행하면 작업 습관으로 바뀝니다.
