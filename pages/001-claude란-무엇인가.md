# 001. Claude란 무엇인가

난이도: 초급  
기준일: 2026년 05월 03일
저자: AI_Innovation_Studio

![Claude 생태계](../assets/02-claude-ecosystem.png)

## 핵심 개념

Claude는 사람의 요청을 이해하고 글쓰기, 분석, 코딩, 요약, 설계, 검토, 자동화 작업을 도와주는 AI 시스템입니다. 하지만 이 책에서 Claude를 단순한 “질문하면 답하는 챗봇”으로만 다루지는 않습니다. 2026년의 Claude 생태계는 대화형 AI, 코드 에이전트, 외부 도구 연결, 업무 자동화, 개발 파이프라인까지 포함하는 작업 환경으로 확장되었습니다.

Claude를 이해할 때 가장 중요한 질문은 “무엇을 알고 있나?”가 아니라 “어떤 환경에서 어떤 행동을 할 수 있나?”입니다. 같은 모델이라도 웹 채팅창에서 쓸 때와 터미널에서 Claude Code로 쓸 때의 능력은 상당히 다르게 느껴집니다. 웹에서는 답변을 생성하지만, Claude Code에서는 파일을 읽고, 코드를 수정하고, 테스트를 실행하고, Git 작업을 도와줄 수 있습니다.

## Claude를 바라보는 세 가지 관점

첫 번째는 대화 파트너입니다. Claude는 글쓰기, 학습, 기획, 요약, 아이디어 확장에 강합니다. 이 단계에서는 사용자가 문장을 입력하고 Claude가 응답합니다.

두 번째는 작업 보조자입니다. Claude는 주어진 자료를 바탕으로 구조화된 문서, 체크리스트, 비교표, 실행 계획을 만들어 줍니다. 이 단계에서는 입력 자료와 출력 형식이 중요합니다.

세 번째는 에이전트입니다. Claude Code처럼 도구와 연결된 Claude는 코드를 읽고 수정하며, 명령을 실행하고, 실패 결과를 보고 다시 고칩니다. 이 단계에서는 권한, 범위, 검증이 중요합니다.

## 왜 중요한가

초급자가 흔히 하는 실수는 Claude를 “똑똑한 검색창”으로만 쓰는 것입니다. 그렇게 쓰면 질문의 품질이 낮을수록 답변의 품질도 흔들립니다. 반대로 Claude를 작업 파트너로 다루면 더 좋은 결과를 얻을 수 있습니다. 목표를 알려주고, 맥락을 제공하고, 제약을 정하고, 완료 기준을 지정하면 Claude는 단순 답변이 아니라 작업 가능한 결과를 만들어 냅니다.

Claude Code에서는 이 차이가 더 큽니다. Claude Code는 실제 프로젝트 파일을 다룰 수 있으므로 요청이 모호하면 원하지 않는 파일을 읽거나, 너무 넓은 범위를 수정하거나, 검증 없이 결과를 제안할 수 있습니다. 따라서 Claude를 잘 쓰는 능력은 곧 “작업을 명확히 정의하는 능력”입니다.

## 실습

Claude에게 아래 세 가지 요청을 각각 해 보세요.

```text
Claude가 무엇인지 초등학생도 이해할 수 있게 설명해줘.
```

```text
Claude를 업무 자동화 관점에서 설명해줘. 예시는 문서, 코드, 이메일, 데이터 분석으로 나눠줘.
```

```text
Claude를 개발자 관점에서 설명해줘. Claude Code, MCP, Skills, Hooks, Agents와 연결해서 설명해줘.
```

세 답변의 차이를 비교하면 Claude가 “누구에게 설명하느냐”와 “어떤 목적이냐”에 따라 상당히 다른 방식으로 답한다는 점을 확인할 수 있습니다.

## Claude 또는 Claude Code에 입력할 프롬프트

```text
나는 Claude를 처음 배우고 있다.
Claude를 단순 챗봇이 아니라 작업 파트너와 에이전트 관점에서 이해하고 싶다.

다음 형식으로 설명해줘.
1. Claude를 한 문장으로 정의
2. Claude가 잘하는 일 10가지
3. Claude가 조심해야 하는 일 5가지
4. Claude Web과 Claude Code의 차이
5. 내가 오늘 바로 해볼 수 있는 실습 3개
```

## 체크리스트

- [ ] Claude를 챗봇, 작업 보조자, 에이전트 관점으로 구분할 수 있다.
- [ ] Claude Web과 Claude Code의 차이를 설명할 수 있다.
- [ ] Claude에게 줄 요청에 목표와 맥락을 포함할 수 있다.
- [ ] Claude의 결과를 그대로 믿지 않고 검토해야 하는 이유를 이해한다.

## 흔한 실수

- “알아서 해줘”처럼 범위가 없는 요청을 한다.
- Claude가 생성한 답변을 검증 없이 복사한다.
- Claude Code에서 권한 승인 의미를 이해하지 않고 진행한다.
- Claude를 검색 엔진처럼만 사용한다.
