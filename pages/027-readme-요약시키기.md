# 027. README 요약시키기

난이도: 초급  
기준일: 2026년 05월 03일
저자: AI_Innovation_Studio

## 핵심 개념

README는 프로젝트의 입구입니다. Claude Code에게 README를 요약하게 하면 설치, 실행, 테스트, 배포, 주의사항을 빠르게 파악할 수 있습니다.

하지만 README 요약은 단순히 짧게 줄이는 일이 아닙니다. 독자에 따라 필요한 정보가 다릅니다. 신규 개발자, 비개발자 기획자, 운영 담당자, 리뷰어에게 필요한 요약은 서로 다릅니다.

## 먼저 독자를 정하기

요약을 요청하기 전에 누가 읽을지 정하세요.

| 독자 | 필요한 요약 |
| --- | --- |
| 신규 개발자 | 설치, 실행, 테스트, 폴더 구조 |
| 프로젝트 매니저 | 목적, 주요 기능, 일정/운영 영향 |
| 운영 담당자 | 배포, 환경 변수, 장애 대응 |
| 코드 리뷰어 | 변경 전 알아야 할 구조와 테스트 |
| 비개발자 | 서비스 목적, 사용 흐름, 주의사항 |

독자를 정하지 않으면 Claude는 일반적인 요약을 만들고, 실제 작업에는 부족할 수 있습니다.

## 좋은 요청

```text
README.md를 신규 개발자 온보딩용으로 요약해줘.

포함할 것:
1. 프로젝트 목적
2. 설치 방법
3. 로컬 실행 방법
4. 테스트 방법
5. 배포 또는 빌드 방법
6. 주의해야 할 환경 변수
7. 처음 봐야 할 폴더와 파일

README에 없는 내용은 추정하지 말고 "README에 없음"이라고 표시해줘.
```

“README에 없는 내용은 추정하지 말라”는 조건이 중요합니다. 문서에 없는 정보를 그럴듯하게 보충하면 오히려 위험합니다.

## README 품질 점검

요약이 끝나면 README 자체의 빈틈도 확인할 수 있습니다.

```text
README를 읽고 신규 개발자가 막힐 만한 부분을 찾아줘.

출력:
- 누락된 정보
- 모호한 명령
- 오래됐을 가능성이 있는 내용
- 보안상 공개하면 안 될 수 있는 정보
- 개선 제안
```

이 요청은 문서를 개선하는 데 특히 유용합니다.

## 실행 명령 검증

README에 적힌 명령은 실제 설정 파일과 맞지 않을 수 있습니다. Claude Code에게 교차 확인을 요청하세요.

```text
README의 실행/테스트/빌드 명령이 설정 파일과 일치하는지 확인해줘.
근거 파일을 함께 표시해줘.
아직 파일은 수정하지 마.
```

예를 들어 README에는 `npm test`라고 되어 있는데 `package.json`에는 `test:unit`만 있을 수 있습니다. 이런 불일치는 초급자에게 큰 장애물이 됩니다.

## 짧은 요약과 작업용 요약

README 요약은 목적에 따라 길이가 달라야 합니다.

짧은 요약:

```text
README를 10줄 이내로 요약해줘.
비개발자도 이해할 수 있게 써줘.
```

작업용 요약:

```text
README를 바탕으로 오늘 개발을 시작하기 위한 체크리스트를 만들어줘.
각 항목에 필요한 명령과 확인 결과를 포함해줘.
```

보고용 요약:

```text
README를 팀 리더에게 공유할 5문장 요약으로 바꿔줘.
기술 세부사항보다 프로젝트 목적과 운영상 주의사항을 우선해줘.
```

## 실습

현재 프로젝트의 README를 대상으로 다음 세 가지 요약을 만들어 보세요.

1. 신규 개발자용 10줄 요약
2. 비개발자용 5문장 요약
3. 실행 준비 체크리스트

그 다음 세 요약에서 빠진 정보가 무엇인지 비교하세요.

## Claude Code에 입력할 프롬프트

```text
README.md를 읽고 신규 팀원 온보딩용으로 요약해줘.

제약:
- README에 없는 내용은 추정하지 말고 "README에 없음"이라고 표시해줘.
- 실행 명령, 테스트 명령, 빌드 명령은 근거 줄이나 근거 파일을 함께 적어줘.
- 비밀정보처럼 보이는 값이 있으면 그대로 반복하지 말고 위험만 설명해줘.

출력 형식:
1. 프로젝트 한 문장 요약
2. 설치 방법
3. 실행 방법
4. 테스트 방법
5. 주요 폴더
6. 환경 변수와 주의사항
7. README에서 보완하면 좋은 점
```

## 체크리스트

- [ ] 요약의 독자를 먼저 정한다.
- [ ] README에 없는 내용을 추정하지 않게 한다.
- [ ] 실행, 테스트, 빌드 명령의 근거를 확인한다.
- [ ] 누락되거나 오래된 문서를 찾는다.
- [ ] 민감 정보가 포함되어 있는지 확인한다.

## 흔한 실수

- “짧게 요약해줘”만 요청한다.
- README 내용과 설정 파일의 불일치를 확인하지 않는다.
- 비개발자용 요약에 지나친 기술 세부사항을 넣는다.
- README에 없는 내용을 사실처럼 보충한다.
