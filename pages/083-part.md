# Part 09. MCP

이 장은 고급 단계 독자를 위한 흐름입니다. 각 페이지는 하나의 주제를 다루며, 개념 설명과 실습 프롬프트를 함께 제공합니다.

## 처음 읽는 분께

이 파트는 용어를 외우는 장이 아니라 실제 작업에서 판단 기준을 세우는 장입니다. 각 항목을 읽을 때는 “언제 쓰는가”, “무엇을 조심해야 하는가”, “어떻게 검증하는가”를 함께 확인하세요.

초보자는 본문을 읽은 뒤 바로 실습 프롬프트를 실행해 보는 것이 좋습니다. 실행 결과가 예상과 다르면 실패로 보지 말고, 어떤 맥락이 부족했는지 기록하세요. 그 기록이 다음 요청을 더 정확하게 만드는 재료가 됩니다.

## 이 장에서 다루는 100선 항목

- 083. MCP 개념: MCP는 Claude가 외부 데이터와 도구에 안전하게 접근하도록 하는 프로토콜이다.
- 084. stdio MCP와 HTTP MCP: 로컬 실행 서버와 원격 HTTP 서버의 차이를 이해한다.
- 085. GitHub MCP: 이슈, PR, 코드 리뷰, 릴리스 작업을 GitHub MCP로 연결한다.
- 086. Database MCP: 데이터베이스 접근은 읽기 전용, 샌드박스, 승인 절차가 중요하다.
- 087. Filesystem MCP: 파일 접근 MCP는 강력하지만 범위 제한이 핵심이다.
- 088. Browser/Chrome MCP: 브라우저 MCP는 웹 앱 검증, 스크린샷, DOM 조사에 유용하다.
- 089. Notion, Jira, Slack 연결 전략: 업무 도구 연결은 편리하지만 권한과 데이터 노출을 설계해야 한다.
- 090. OAuth와 인증 보안: 토큰 저장, 권한 범위, step-up auth, 철회 절차를 이해한다.
- 091. MCP 서버 검증 체크리스트: 알 수 없는 MCP는 코드, 권한, 네트워크, 업데이트 정책을 검토한 뒤 사용한다.

## 학습 방식

각 항목을 읽은 뒤 바로 Claude 또는 Claude Code에 실습 프롬프트를 입력해 보세요. 읽기만 하면 기능 목록으로 남고, 직접 실행하면 작업 습관으로 바뀝니다.
