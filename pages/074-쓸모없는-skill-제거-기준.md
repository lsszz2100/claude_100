# 074. 쓸모없는 Skill 제거 기준

난이도: 고급  
기준일: 2026년 05월 03일

## 핵심 개념

Skill은 많을수록 좋은 것이 아닙니다. 설명이 모호하거나, 거의 쓰이지 않거나, 오래된 절차를 담고 있거나, 위험한 도구 권한을 가진 Skill은 작업 품질을 떨어뜨릴 수 있습니다.

Skill 운영에는 추가만큼 제거 기준이 필요합니다.

## 제거 후보

| 후보 | 이유 |
| --- | --- |
| description이 모호한 Skill | 잘못 활성화되거나 사용되지 않음 |
| 너무 넓은 Skill | 하나의 Skill이 여러 일을 하며 예측 불가 |
| 오래된 명령을 담은 Skill | 잘못된 작업 유도 |
| 쓰이지 않는 Skill | 관리 비용만 증가 |
| 위험한 allowed-tools | 필요 이상 권한 부여 |
| CLAUDE.md와 중복 | 맥락 낭비와 충돌 |

## 점검 프롬프트

```text
현재 프로젝트 Skills를 점검해줘.

확인할 것:
1. 각 Skill의 목적
2. description의 명확성
3. CLAUDE.md 또는 다른 Skill과의 중복
4. 오래된 명령이나 파일 경로
5. allowed-tools가 과도한지
6. 제거, 유지, 수정 후보

아직 파일은 삭제하지 마.
```

## 수정할 Skill

다음은 제거보다 수정이 낫습니다.

- 목적은 명확하지만 description이 약하다.
- 좋은 절차가 있지만 출력 형식이 없다.
- reference 분리가 안 되어 너무 길다.
- 도구 권한만 과도하다.

## 유지할 Skill

다음 조건을 만족하면 유지할 가치가 있습니다.

- 한 달에 여러 번 반복되는 작업을 다룬다.
- 팀 기준을 일관되게 적용한다.
- 출력 형식이 안정적이다.
- 위험한 권한 없이도 효과가 있다.
- 새 팀원이 읽어도 사용 시점이 분명하다.

## 제거할 Skill

다음은 제거가 낫습니다.

- 더 이상 존재하지 않는 도구나 명령을 전제로 한다.
- 다른 Skill과 거의 중복된다.
- 팀 표준과 충돌한다.
- 위험한 명령 실행을 권장한다.
- 실제 사용 사례가 없다.

## deprecated 처리

바로 삭제하기 애매한 Skill은 먼저 deprecated 표시를 붙입니다.

```markdown
# Deprecated

이 Skill은 더 이상 새 작업에 사용하지 않습니다.
대체 Skill: `skills/review`
삭제 예정일: 2026-06-30
```

팀 문서나 다른 Skill에서 참조가 사라진 뒤 삭제하면 혼란을 줄일 수 있습니다.

## 제거 전 확인

```text
이 Skill을 삭제하기 전에 영향 범위를 확인해줘.

확인할 것:
- README나 CLAUDE.md에서 참조하는지
- 다른 Skill에서 참조하는지
- 팀 문서에 사용법이 남아 있는지
- 대체 Skill이 있는지
- 삭제 대신 deprecated 표시가 나은지
```

## 체크리스트

- [ ] Skill 목록을 주기적으로 검토한다.
- [ ] description이 구체적인지 확인한다.
- [ ] 오래된 명령과 경로를 찾는다.
- [ ] 도구 권한이 최소인지 확인한다.
- [ ] 삭제 전 참조 관계를 확인한다.
- [ ] 바로 삭제하기 어려우면 deprecated 기간을 둔다.
- [ ] 유지할 Skill의 사용 기준을 문서화한다.
