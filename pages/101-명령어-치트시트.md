# 101. 명령어 치트시트

저자: AI_Innovation_Studio

이 부록은 Claude Code를 사용할 때 자주 확인하는 명령과 요청 문장을 모은 빠른 참조표입니다. 명령은 버전과 설정에 따라 달라질 수 있으므로, 실제 환경에서는 `/help`와 `/doctor`로 한 번 더 확인하세요.

## Claude Code 기본 명령

```text
/help       사용 가능한 명령 확인
/clear      현재 대화 기록 비우기
/compact    긴 대화를 요약해 이어가기
/cost       현재 세션 사용량 확인
/model      모델 확인 또는 변경
/config     설정 확인 또는 변경
/status     계정과 시스템 상태 확인
/doctor     설치 상태 점검
```

## 프로젝트 메모리

```text
/init       CLAUDE.md 초안 생성
/memory     메모리 파일 확인 또는 편집
```

## IDE와 확장

```text
/ide        외부 터미널 세션을 IDE와 연결
/mcp        MCP 서버 연결과 인증 관리
/agents     Subagents 생성, 수정, 삭제
/review     코드 리뷰 요청
```

## 자주 쓰는 안전 요청

```text
아직 파일은 수정하지 말고 먼저 분석과 계획만 작성해줘.
```

```text
현재 변경사항을 git diff 기준으로 요약하고, 위험한 변경과 테스트 필요 항목을 알려줘.
```

```text
수정 후 실행한 테스트 명령, 결과, 남은 위험을 요약해줘.
```

```text
내가 승인하기 전에는 설치, 삭제, 배포, 외부 전송 명령을 실행하지 마.
```

## 위험 작업 요청

```text
이 작업에 삭제, 이동, 설치, 배포, 외부 전송, 권한 변경이 필요한지 먼저 분류해줘.
필요하다면 실행 명령과 영향 범위를 보여주고, 내 승인 전에는 실행하지 마.
```

```text
실행 전에 대상 파일의 절대 경로, 개수, 변경 전/후를 먼저 보여줘.
대량 변경은 샘플 3개로 dry run한 뒤 진행해줘.
```

## Git 확인

```text
git status
git diff
git diff --stat
git log --oneline -10
```

## 상황별 첫 요청

```text
이 프로젝트의 구조, 실행 방법, 테스트 방법, 주의할 파일을 먼저 요약해줘.
아직 수정하지 말고 읽기만 해줘.
```

```text
이 오류를 재현 가능한 정보 기준으로 분석해줘.
가능한 원인을 우선순위로 정리하고, 가장 작은 확인 명령부터 제안해줘.
```

```text
현재 변경사항을 리뷰어 관점으로 검토해줘.
버그, 보안 위험, 테스트 누락, 문서 누락을 심각도 순서로 알려줘.
```

```text
긴 작업을 이어갈 수 있게 현재 목표, 완료한 일, 남은 일, 위험, 다음 명령을 요약해줘.
```

## 이어가기 요청

```text
이전 세션을 이어가자.
먼저 git status, 최근 변경 파일, 실패한 검증, 다음 작업 후보를 확인하고
파일 수정 전에 무엇을 이어갈지 짧게 설명해줘.
```

## 작업 종료 전 점검

- [ ] 수정된 파일 목록을 확인했다.
- [ ] 테스트 또는 검증 명령을 실행했다.
- [ ] 실행하지 못한 검증을 명시했다.
- [ ] 다음 세션에서 이어갈 수 있게 남은 TODO를 남겼다.
- [ ] 민감 정보가 diff나 로그에 남지 않았는지 확인했다.
- [ ] 위험 작업이 있었다면 승인 과정과 영향 범위를 남겼다.
