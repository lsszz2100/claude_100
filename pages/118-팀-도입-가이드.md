# 118. 팀 도입 가이드

Claude Code를 팀에 도입할 때는 “모두 쓰게 하기”보다 “작고 안전한 반복 작업 하나를 표준화하기”가 먼저입니다. 팀 규칙, 권한, 검토 기준이 없으면 생산성보다 혼란이 커질 수 있습니다.

## 도입 단계

1. 개인 실험
2. 작은 팀 파일럿
3. 공통 `CLAUDE.md` 작성
4. 리뷰와 테스트 기준 합의
5. Skills 또는 Hooks 일부 도입
6. MCP와 외부 도구 연결 검토
7. 운영 표준 문서화

## 먼저 표준화할 작업

- PR 리뷰 초안
- 테스트 실패 로그 분석
- README와 운영 문서 갱신
- 릴리스 노트 초안
- 반복 체크리스트 생성

## 팀 규칙 예시

- 운영 배포 명령은 자동 실행하지 않는다.
- secret, 고객 데이터, 내부 토큰은 프롬프트에 넣지 않는다.
- 대규모 변경은 계획 승인 후 진행한다.
- Claude가 작성한 코드는 사람이 리뷰한다.
- 테스트 결과와 미실행 검증을 PR에 남긴다.

## Claude에게 줄 요청

```text
우리 팀에 Claude Code를 도입하기 위한 4주 파일럿 계획을 작성해줘.

팀 상황:
- 인원:
- 기술 스택:
- 반복 업무:
- 보안 제약:

출력:
1. 주차별 목표
2. 사용할 기능
3. 금지할 사용 방식
4. 성공 지표
5. 팀 문서 초안 목차
```

## 체크리스트

- [ ] 도입 목적이 명확하다.
- [ ] 보안과 권한 기준이 있다.
- [ ] 성공 지표를 정했다.
- [ ] 팀원이 거절하거나 중단할 기준도 정했다.
