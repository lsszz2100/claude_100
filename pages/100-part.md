# Part 11. 프로덕션 운영

이 장은 고급 단계 독자를 위한 흐름입니다. 앞의 장들이 Claude Code의 기능과 확장 방법을 다뤘다면, 이 장은 그것을 팀과 운영 환경에 넣을 때 필요한 기준을 정리합니다.

프로덕션 운영에서 중요한 것은 “AI가 코드를 만들 수 있는가”가 아니라 “그 결과를 검증하고, 배포하고, 문제가 생겼을 때 되돌릴 수 있는가”입니다. 권한, 로그, 리뷰, 테스트, 승인 절차가 함께 설계되어야 합니다.

## 처음 읽는 분께

이 파트는 용어를 외우는 장이 아니라 실제 작업에서 판단 기준을 세우는 장입니다. 각 항목을 읽을 때는 “언제 쓰는가”, “무엇을 조심해야 하는가”, “어떻게 검증하는가”를 함께 확인하세요.

초보자는 본문을 읽은 뒤 바로 실습 프롬프트를 실행해 보는 것이 좋습니다. 실행 결과가 예상과 다르면 실패로 보지 말고, 어떤 맥락이 부족했는지 기록하세요. 그 기록이 다음 요청을 더 정확하게 만드는 재료가 됩니다.

## 이 장에서 다루는 100선 항목

- 100. Claude Code 프로덕션 운영 전략: AI 개발 결과물을 운영하려면 테스트, 보안, 배포, 모니터링, 피드백 루프가 필요하다.

## 이 파트의 목표

이 파트는 책의 앞부분에서 배운 기능을 운영 기준으로 묶는 장입니다. Claude Code로 만든 결과물이 실제 사용자에게 영향을 주려면 코드 작성보다 검증, 승인, 배포, 관찰, 복구가 더 중요해집니다.

이 파트를 읽은 뒤에는 다음 운영 기준을 점검할 수 있어야 합니다.

- 어떤 테스트가 배포 전 필수인지 정한다.
- 보안 리뷰와 권한 검토가 필요한 변경을 구분한다.
- 배포 실패 시 되돌리는 절차를 문서화한다.
- 비용, 지연 시간, 오류율, 사용자 피드백을 관찰한다.
- 프롬프트와 모델 변경도 운영 변경으로 다룬다.

## 추천 실습 순서

현재 프로젝트 하나를 골라 출시 전 점검표를 만드세요. 테스트, 보안, 배포, 모니터링, 롤백, 비용 기준을 한 장에 모으면 Claude Code 작업 결과를 개인 생산성에서 운영 가능한 시스템으로 옮길 수 있습니다.

## 학습 방식

이 장을 읽은 뒤에는 현재 팀 또는 개인 프로젝트의 운영 준비 상태를 점검해 보세요. `CLAUDE.md`, MCP 권한, Hooks, Subagents, CI/CD, 배포 승인 절차를 하나의 체크리스트로 묶으면 Claude Code를 개인 생산성 도구에서 반복 가능한 개발 시스템으로 확장할 수 있습니다.
