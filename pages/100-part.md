# Part 11. 프로덕션 운영

이 장은 고급 단계 독자를 위한 흐름입니다. 앞의 장들이 Claude Code의 기능과 확장 방법을 다뤘다면, 이 장은 그것을 팀과 운영 환경에 넣을 때 필요한 기준을 정리합니다.

프로덕션 운영에서 중요한 것은 “AI가 코드를 만들 수 있는가”가 아니라 “그 결과를 검증하고, 배포하고, 문제가 생겼을 때 되돌릴 수 있는가”입니다. 권한, 로그, 리뷰, 테스트, 승인 절차가 함께 설계되어야 합니다.

## 이 장에서 다루는 100선 항목

- 100. Claude Code 프로덕션 운영 전략: AI 개발 결과물을 운영하려면 테스트, 보안, 배포, 모니터링, 피드백 루프가 필요하다.

## 학습 방식

이 장을 읽은 뒤에는 현재 팀 또는 개인 프로젝트의 운영 준비 상태를 점검해 보세요. `CLAUDE.md`, MCP 권한, Hooks, Subagents, CI/CD, 배포 승인 절차를 하나의 체크리스트로 묶으면 Claude Code를 개인 생산성 도구에서 반복 가능한 개발 시스템으로 확장할 수 있습니다.
