# 055. 반복 작업 자동화

난이도: 중급  
기준일: 2026년 05월 03일
저자: AI_Innovation_Studio

## 핵심 개념

반복 작업 자동화는 “귀찮은 일을 모두 자동화”하는 것이 아닙니다. 기준이 명확하고, 입력과 출력이 안정적이며, 실패했을 때 되돌릴 수 있는 작업부터 자동화해야 합니다.

Claude Code는 반복 작업을 분석하고, 자동화 후보를 분류하고, 스크립트나 체크리스트 초안을 만드는 데 유용합니다.

## 자동화 후보 판단

좋은 자동화 후보:

- 매주 또는 매일 반복한다.
- 입력 파일이나 명령이 일정하다.
- 성공/실패 기준이 명확하다.
- 사람이 매번 같은 판단을 한다.
- 실패해도 되돌릴 수 있다.

나쁜 자동화 후보:

- 요구사항이 매번 크게 바뀐다.
- 사람의 복잡한 판단이 핵심이다.
- 운영 데이터 삭제처럼 실패 비용이 크다.
- 입력 형식이 불안정하다.
- 한 번만 할 작업이다.

## 분석 요청

```text
아래 반복 작업을 자동화할 수 있는지 분석해줘.

현재 사람이 하는 절차:
1. [단계]
2. [단계]
3. [단계]

요청:
1. 자동화하기 좋은 부분
2. 자동화하면 위험한 부분
3. 필요한 입력
4. 생성할 출력
5. 실패 시 되돌리는 방법
6. 첫 번째 작은 자동화 범위

아직 스크립트는 만들지 마.
```

## 자동화 설계 템플릿

```markdown
# Automation Plan: [작업 이름]

## Goal

- [자동화 목표]

## Inputs

- [입력 파일/명령/환경 변수]

## Outputs

- [생성 파일/요약/변경사항]

## Steps

1. [단계]
2. [단계]
3. [단계]

## Safety

- Dry run:
- Backup:
- Confirmation:
- Rollback:

## Validation

- [성공 확인 방법]

## First Version Scope

- [처음 자동화할 최소 범위]
```

## dry run 먼저 만들기

파일을 이동하거나 수정하는 자동화는 dry run이 중요합니다.

```text
이 자동화 스크립트는 먼저 dry run 모드로 설계해줘.

dry run에서는:
- 실제 파일을 수정하지 않는다.
- 변경 예정 목록만 출력한다.
- 위험한 파일을 표시한다.
- 사용자가 승인할 수 있게 요약한다.
```

## 예시: 주간 보고서 정리

```text
downloads 폴더에서 이번 주 CSV 파일을 찾아
reports/YYYY-WW/ 폴더로 복사하고,
파일명을 날짜_고객사_리포트.csv로 표준화하고 싶다.

제약:
- 처음에는 dry run만 만든다.
- 원본 파일은 삭제하지 않는다.
- 파일명 충돌이 있으면 중단한다.
- 처리 결과를 report-log.md에 기록한다.
```

이런 요청은 입력, 출력, 안전장치가 분명합니다.

## 자동화 구현 승인

```text
좋아. dry run 버전만 구현해줘.

제약:
- 실제 파일 수정, 이동, 삭제는 하지 마.
- 변경 예정 목록을 표로 출력하게 해줘.
- 충돌과 누락을 감지하게 해줘.
- 샘플 폴더에서만 테스트해줘.
```

첫 버전은 작고 안전해야 합니다.

## 실습

자신이 자주 하는 작업 하나를 골라 자동화 후보 분석표를 만들어 보세요.

예시:

- 매주 회의록 요약
- 테스트 결과 정리
- 릴리스 노트 초안 작성
- 이미지 파일 이름 정리
- CSV 검증

## Claude Code에 입력할 프롬프트

```text
다음 반복 작업을 자동화 후보로 분석해줘.

작업 설명:
[사람이 현재 하는 일]

입력:
[파일/폴더/명령]

출력:
[원하는 결과]

제약:
- 아직 스크립트는 만들지 마.
- 자동화하기 좋은 부분과 위험한 부분을 나눠줘.
- dry run 설계를 포함해줘.
- 첫 번째 버전은 가장 작은 범위로 제안해줘.
- 실패했을 때 되돌릴 방법을 포함해줘.
```

## 체크리스트

- [ ] 반복 빈도와 절차가 명확하다.
- [ ] 입력과 출력이 안정적이다.
- [ ] dry run이 가능하다.
- [ ] 실패 시 되돌릴 수 있다.
- [ ] 첫 버전 범위가 작다.

## 흔한 실수

- 한 번만 할 일을 자동화한다.
- 삭제나 이동을 dry run 없이 실행한다.
- 실패 조건을 설계하지 않는다.
- 사람의 판단이 필요한 부분까지 억지로 자동화한다.
