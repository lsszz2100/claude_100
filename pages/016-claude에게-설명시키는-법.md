# 016. Claude에게 설명시키는 법

난이도: 초급  
기준일: 2026년 05월 03일

## 핵심 개념

Claude는 설명을 잘합니다. 하지만 “설명해줘”라고만 하면 독자 수준, 설명 깊이, 예시 방식이 매번 달라집니다. 좋은 설명을 얻으려면 누구에게 설명하는지, 어떤 목적의 설명인지, 어느 정도 깊이가 필요한지 지정해야 합니다.

## 설명의 세 가지 목적

### 1. 학습용 설명

처음 배우는 사람에게 개념을 이해시키는 설명입니다.

```text
초급자가 이해할 수 있게 비유와 예시를 들어 설명해줘.
전문 용어는 처음 나올 때 풀어서 설명해줘.
```

### 2. 코드 이해용 설명

코드가 어떻게 동작하는지 파악하기 위한 설명입니다.

```text
이 파일의 역할을 설명해줘.
함수별 책임, 데이터 흐름, 외부 의존성을 나눠서 정리해줘.
```

### 3. 의사결정용 설명

여러 선택지 중 하나를 고르기 위한 설명입니다.

```text
이 설계 선택의 장단점과 유지보수 영향을 설명해줘.
단기 효과와 장기 비용을 구분해줘.
```

## 좋은 설명 요청의 구조

```text
대상 독자:
설명 목적:
설명 깊이:
포함할 내용:
제외할 내용:
출력 형식:
```

## 예시

```text
대상 독자:
React는 조금 알지만 인증 흐름은 잘 모르는 초급 개발자

설명 목적:
이 프로젝트의 로그인 흐름을 이해하고 싶다.

설명 깊이:
코드 줄 단위가 아니라 파일과 함수 역할 중심으로 설명해줘.

포함할 내용:
관련 파일, 데이터 흐름, 토큰 저장 위치, 실패 처리 방식

출력 형식:
1. 전체 요약
2. 파일별 역할
3. 로그인 성공 흐름
4. 로그인 실패 흐름
5. 주의할 점
```

## Claude Code에서 활용하기

Claude Code는 실제 파일을 읽을 수 있으므로 설명 요청이 매우 유용합니다. 처음 보는 프로젝트에서는 수정 요청보다 설명 요청을 먼저 해야 합니다. 설명을 통해 프로젝트 구조를 이해하면 이후 요청의 품질이 올라갑니다.

## 실습

현재 프로젝트에서 핵심 파일 하나를 고른 뒤 이렇게 요청해 보세요.

```text
이 파일을 초급 개발자에게 설명하듯 정리해줘.

다음 형식으로 작성해줘.
1. 이 파일의 목적
2. 중요한 함수 또는 클래스
3. 입력과 출력
4. 다른 파일과의 관계
5. 수정할 때 주의할 점
```

## 설명 검증하기

Claude의 설명도 틀릴 수 있습니다. 설명을 검증하려면 이렇게 물어보세요.

```text
방금 설명에서 실제 코드로 확인 가능한 근거를 파일명과 함수명 기준으로 표시해줘.
추측한 부분이 있다면 따로 분리해줘.
```

## 체크리스트

- [ ] 설명 대상 독자를 지정했다.
- [ ] 설명 목적을 지정했다.
- [ ] 포함할 내용과 제외할 내용을 정했다.
- [ ] 실제 코드 근거를 요구했다.
- [ ] 추측과 확인된 사실을 구분하게 했다.

## 흔한 실수

- “자세히 설명해줘”라고만 요청한다.
- 독자 수준을 지정하지 않는다.
- 코드 근거 없이 설명을 믿는다.
- 설명을 받은 뒤 문서화하지 않는다.
