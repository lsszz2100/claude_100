# 109. 추천 Hook 템플릿

저자: AI_Innovation_Studio

Hook은 Claude Code의 특정 이벤트에 자동 실행되는 명령입니다. 자동 실행은 편리하지만 위험하므로, 처음에는 읽기 전용 검증이나 로그처럼 영향이 작은 용도로 시작하세요.

초보자는 Hook을 “Claude가 무언가를 하려고 할 때 자동으로 실행되는 안전장치”로 이해하면 됩니다. 다만 자동 실행이라는 특성 때문에 잘못 설계하면 작업을 방해하거나 민감 정보를 로그로 남길 수 있습니다.

처음 만드는 Hook은 차단보다 관찰에 가깝게 설계하는 편이 좋습니다. 예를 들어 파일 수정 후 포맷 상태를 확인하거나, 위험 파일 수정 시 경고 메시지를 출력하는 정도에서 시작하세요.

## 설계 원칙

- 자동 실행 범위를 작게 유지한다.
- 삭제, 배포, 외부 전송 같은 명령은 기본적으로 피한다.
- 입력값을 신뢰하지 말고 검증한다.
- 실패 메시지는 짧고 구체적으로 만든다.
- 팀원이 끌 수 있는 방법을 문서화한다.

## settings.json 예시

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "python .claude/hooks/check_format.py"
          }
        ]
      }
    ]
  }
}
```

## Hook 스크립트 기본 구조

```python
import json
import sys

def main():
    payload = json.load(sys.stdin)

    return 0

if __name__ == "__main__":
    raise SystemExit(main())
```

## Claude에게 줄 요청

```text
다음 자동화 후보를 Claude Code Hook으로 만들지 검토해줘.

후보:
[설명]

검토 기준:
1. 자동 실행이 필요한가?
2. 수동 명령이나 Skill로 충분한가?
3. 권한과 보안 위험은 무엇인가?
4. 최소 권한 설계는 무엇인가?
```

## 작성 순서

1. 자동 실행이 꼭 필요한지 먼저 판단합니다.
2. Hook이 실행되는 시점을 정합니다. 수정 전인지, 수정 후인지, 명령 실행 전인지에 따라 위험이 달라집니다.
3. Hook 스크립트가 읽는 입력값을 검증합니다.
4. 실패했을 때 작업을 중단할지, 경고만 남길지 결정합니다.
5. 팀원이 끄는 방법과 실패 시 확인할 로그 위치를 문서화합니다.

## 위험한 Hook의 신호

- 삭제, 배포, 외부 전송을 자동으로 실행한다.
- 실패 이유를 자세히 남기지 않는다.
- 민감 파일 내용을 로그에 그대로 출력한다.
- 모든 프로젝트에 같은 규칙을 강제로 적용한다.

## 체크리스트

- [ ] Hook이 꼭 필요한 자동화다.
- [ ] 실패해도 작업 파일을 망가뜨리지 않는다.
- [ ] 민감 정보가 로그에 남지 않는다.
- [ ] 팀에 비활성화 방법을 공유했다.
