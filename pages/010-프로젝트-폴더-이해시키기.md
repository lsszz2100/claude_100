# 010. 프로젝트 폴더 이해시키기

난이도: 초급  
기준일: 2026년 05월 03일

## 핵심 개념

Claude Code는 프로젝트 폴더 안에서 실행될 때 가장 강력합니다. 하지만 “폴더 안에 있다”는 사실만으로 프로젝트를 충분히 이해하는 것은 아닙니다. 사용자는 Claude가 어떤 파일을 우선 읽어야 하는지, 어떤 관점으로 분석해야 하는지, 어떤 결과 형식으로 보고해야 하는지 알려줘야 합니다.

## Claude가 먼저 봐야 할 파일

프로젝트마다 다르지만 일반적으로 다음 파일이 중요합니다.

- README.md
- package.json, pyproject.toml, build.gradle, pom.xml 같은 빌드/패키지 파일
- Dockerfile, docker-compose.yml
- .env.example
- 테스트 설정 파일
- src 또는 app 폴더
- docs 또는 architecture 문서
- CLAUDE.md가 있다면 프로젝트 지침

Claude에게 “알아서 봐”라고 하는 것보다 “실행과 테스트 방법을 찾는 데 필요한 파일을 우선 확인해”라고 말하는 편이 좋습니다.

## 좋은 분석 요청

```text
이 프로젝트의 구조를 분석해줘.
특히 다음 질문에 답하기 위해 필요한 파일을 우선 읽어줘.

1. 어떤 앱인가?
2. 어떤 기술 스택인가?
3. 로컬 실행 명령은 무엇인가?
4. 테스트 명령은 무엇인가?
5. 배포 관련 파일이 있는가?
6. 주요 기능은 어느 폴더에 있는가?
7. 변경 전에 조심해야 할 파일은 무엇인가?

아직 파일은 수정하지 마.
```

## 분석 결과를 더 좋게 만드는 추가 요청

첫 분석 결과가 나온 뒤에는 이렇게 이어갈 수 있습니다.

```text
방금 분석한 내용을 바탕으로 신규 개발자 온보딩 문서처럼 다시 정리해줘.
다음 형식을 사용해줘.

1. 프로젝트 한 줄 설명
2. 로컬 실행 방법
3. 테스트 방법
4. 핵심 폴더
5. 자주 수정할 파일
6. 수정 전 주의사항
7. 첫 기여 추천 작업
```

이런 방식으로 Claude의 분석 결과를 문서화하면 나중에 `CLAUDE.md`나 `README.md`로 옮길 수 있습니다.

## 실습

현재 프로젝트에서 Claude Code에게 다음을 요청해 보세요.

```text
이 프로젝트의 파일 구조를 보고 Mermaid 없이 텍스트 트리로 요약해줘.
각 폴더가 어떤 역할을 하는지도 한 줄씩 설명해줘.
```

그 다음 이렇게 물어보세요.

```text
이 프로젝트를 Claude Code와 함께 개발하기 위해 CLAUDE.md에 넣어야 할 내용 초안을 작성해줘.
```

## 체크리스트

- [ ] Claude가 먼저 읽어야 할 파일 후보를 알고 있다.
- [ ] 프로젝트 분석 요청에서 “수정 금지”를 명시할 수 있다.
- [ ] 실행 명령과 테스트 명령을 찾도록 요청할 수 있다.
- [ ] 분석 결과를 README나 CLAUDE.md 초안으로 바꿀 수 있다.

## 흔한 실수

- 프로젝트 루트가 아닌 엉뚱한 폴더에서 Claude Code를 실행한다.
- README만 보고 전체 구조를 판단한다.
- 테스트 명령을 확인하지 않고 수정부터 시작한다.
- 분석 결과를 문서로 남기지 않는다.

## 다음 단계

이제 Claude와 Claude Code의 큰 그림을 잡았습니다. 다음 파트에서는 좋은 질문과 프롬프트의 기본기를 배우며, Claude에게 더 안정적인 결과를 끌어내는 법을 익힙니다.
