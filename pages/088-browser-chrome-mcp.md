# 088. Browser/Chrome MCP

난이도: 고급  
기준일: 2026년 05월 03일

## 핵심 개념

Browser 또는 Chrome MCP는 Claude가 웹 페이지를 열고, 클릭하고, 입력하고, 스크린샷을 확인하는 식으로 브라우저 기반 검증을 할 수 있게 합니다. 프론트엔드 작업에서는 코드만 보는 것보다 실제 렌더링을 확인하는 것이 중요합니다.

## 사용 사례

- UI 변경 후 화면 깨짐 확인
- 로그인 흐름 수동 검증
- 접근성 기본 확인
- 반응형 레이아웃 확인
- 콘솔 오류 확인
- E2E 테스트 실패 재현

## 프론트엔드 검증 요청

```text
Browser MCP로 로컬 앱을 확인해줘.

URL:
http://localhost:3000

확인할 것:
1. 주요 화면이 로드되는지
2. 콘솔 오류가 있는지
3. 버튼 클릭과 폼 입력이 동작하는지
4. 모바일 폭에서 텍스트가 겹치지 않는지
5. 접근성상 명확한 문제

아직 코드는 수정하지 말고 관찰 결과만 정리해줘.
```

## 민감 정보 주의

브라우저 도구는 실제 계정, 쿠키, 세션을 다룰 수 있습니다. 운영 계정보다 테스트 계정을 사용하고, 결제/삭제/발송 같은 행동은 차단해야 합니다.

금지 조건 예:

```text
금지:
- 실제 결제
- 실제 이메일 발송
- 사용자 삭제
- 운영 데이터 변경
- 개인정보 화면 캡처 공유
```

## 검증 절차

프론트엔드 검증은 관찰과 수정을 분리해야 합니다.

1. URL과 계정을 확인한다.
2. 주요 화면이 렌더링되는지 본다.
3. 콘솔 오류와 네트워크 실패를 기록한다.
4. 핵심 사용자 흐름을 클릭한다.
5. 모바일 폭과 데스크톱 폭을 확인한다.
6. 관찰 결과를 파일/화면/재현 단계로 요약한다.
7. 수정이 필요하면 별도 계획을 만든다.

## 관찰 결과 형식

```text
Page:
Viewport:
Steps:
Observed:
Expected:
Console errors:
Network failures:
Screenshot note:
Suggested fix area:
```

## 실수하기 쉬운 점

- 테스트 계정이 아니라 실제 계정으로 확인한다.
- UI 관찰 중 바로 코드를 수정해 원인 기록을 잃는다.
- 데스크톱만 보고 모바일 폭을 확인하지 않는다.
- 콘솔 오류를 사용자 영향과 연결하지 않는다.
- 실제 결제, 발송, 삭제 버튼을 누른다.

## 체크리스트

- [ ] 테스트 URL과 계정을 명확히 한다.
- [ ] destructive 사용자 행동을 금지한다.
- [ ] 콘솔 오류와 네트워크 실패를 확인한다.
- [ ] 모바일/데스크톱 폭을 모두 본다.
- [ ] 관찰과 코드 수정을 분리한다.
