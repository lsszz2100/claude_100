# 058. Claude Code Desktop 활용

난이도: 중급  
기준일: 2026년 05월 03일
저자: AI_Innovation_Studio

## 핵심 개념

Claude Desktop은 대화, 문서 검토, 아이디어 정리, MCP 연결 같은 작업에 강점이 있습니다. Claude Code 터미널 작업과 함께 쓰면 “설계와 검토는 Desktop, 실제 코드 수정과 테스트는 Claude Code”처럼 역할을 나눌 수 있습니다.

Desktop을 코드 수정 도구로만 보지 말고, 긴 요구사항 정리와 리뷰 보조 화면으로 활용하세요.

## 잘 맞는 작업

| 작업 | Desktop 활용 방식 |
| --- | --- |
| PRD 초안 | 긴 요구사항을 대화로 정리 |
| 리뷰 기준 작성 | 보안/성능/테스트 체크리스트 생성 |
| 회의록 요약 | 개발 TODO로 변환 |
| 아키텍처 설명 | 코드 수정 전 개념 정리 |
| MCP 기반 문서 조회 | Notion, Drive, Slack 등 업무 맥락 확인 |

실제 파일 수정, 테스트 실행, Git diff 확인은 Claude Code 터미널에서 하는 편이 명확합니다.

## Desktop에서 Code로 넘기는 프롬프트

Desktop에서 요구사항을 정리한 뒤 Claude Code에 넘길 수 있게 다음 형식으로 정리하게 합니다.

```text
이 대화를 Claude Code에 넘길 작업 지시서로 정리해줘.

포함할 것:
1. 목표
2. 배경
3. 범위
4. 비범위
5. 수용 기준
6. 관련 파일 후보
7. 구현 전 확인 질문
8. 테스트 전략
```

그 결과를 Claude Code 터미널에 붙여 넣고, 먼저 분석만 요청합니다.

```text
아래 작업 지시서를 기준으로 현재 코드베이스를 분석해줘.
아직 파일은 수정하지 말고 관련 파일, 구현 계획, 위험, 테스트 방법만 작성해줘.
```

## Code 결과를 Desktop에서 검토하기

Claude Code가 만든 변경 요약을 Desktop에서 리뷰용 문서로 바꿀 수도 있습니다.

```text
아래 변경 요약을 팀 리뷰용 설명으로 바꿔줘.

포함할 것:
- 변경 목적
- 사용자 영향
- 테스트 결과
- 남은 위험
- 리뷰어가 확인할 질문
```

## 주의할 점

Desktop 대화와 Claude Code 세션은 항상 같은 파일 상태를 공유한다고 가정하면 안 됩니다. Desktop에서 정리한 요구사항은 문서일 뿐이고, 실제 구현 전에는 Claude Code가 현재 저장소를 다시 읽어야 합니다.

```text
Desktop에서 정리한 내용은 참고만 하고,
현재 저장소 파일을 다시 확인한 뒤 계획을 세워줘.
```

## 체크리스트

- [ ] Desktop은 요구사항 정리와 검토에 활용한다.
- [ ] 코드 수정 전 Claude Code가 현재 파일을 다시 읽게 한다.
- [ ] Desktop 대화 내용을 작업 지시서 형식으로 정리한다.
- [ ] 실제 테스트와 Git diff 확인은 Claude Code에서 한다.
- [ ] 민감 정보는 Desktop과 Code 모두에서 노출하지 않는다.
