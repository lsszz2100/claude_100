# 059. Claude Code Web 활용

난이도: 중급  
기준일: 2026년 05월 03일
저자: AI_Innovation_Studio

## 핵심 개념

Claude Code on the web은 브라우저에서 Claude Code 작업을 클라우드 인프라에 맡기는 방식입니다. 2026년 05월 03일 기준 연구 프리뷰로 제공되며, GitHub 저장소를 연결해 비동기 작업을 실행하고 브라우저나 모바일에서 진행을 확인할 수 있습니다.

로컬 Claude Code가 “내 컴퓨터에서 즉시 조종하는 작업”이라면, Web은 “잘 정의된 작업을 맡겨두고 결과를 검토하는 작업”에 가깝습니다.

## 잘 맞는 작업

| 작업 | 이유 |
| --- | --- |
| 작은 버그 수정 | 요구사항이 명확하고 테스트로 검증 가능 |
| 문서 보강 | 파일 범위가 명확함 |
| 테스트 추가 | 실패/성공 기준이 분명함 |
| 병렬 조사 | 여러 이슈를 동시에 맡기기 좋음 |
| 로컬에 없는 저장소 질문 | GitHub 연결 기반으로 시작 가능 |

반대로 자주 개입해야 하는 UI 세부 조정, 로컬 장비 의존 작업, 비밀정보가 필요한 작업은 신중해야 합니다.

## Web 작업 지시 템플릿

```text
목표:
[작업 목표]

저장소:
[GitHub repo]

범위:
- [수정할 가능성이 있는 폴더/파일]

비범위:
- [하지 않을 일]

검증:
- [실행할 테스트 또는 확인 방법]

제약:
- 새 의존성은 추가하지 않는다.
- 민감 파일은 읽거나 출력하지 않는다.
- 변경 후 PR 설명에 테스트 결과를 포함한다.
```

Web에서는 작업을 맡긴 뒤 즉시 세부 조종하기 어렵기 때문에 범위와 완료 기준이 더 중요합니다.

## 좋은 Web 작업 예시

```text
README의 로컬 실행 가이드가 package.json과 맞는지 확인하고 수정해줘.

범위:
- README.md
- package.json

검증:
- README에 적은 명령이 package.json scripts에 존재하는지 확인

비범위:
- 앱 코드는 수정하지 않음
- 새 패키지 설치 없음
```

## 터미널로 이어가기

Claude Code Web의 세션은 필요하면 로컬 터미널로 이어갈 수 있습니다. 공식 문서에서는 웹과 터미널 간 이동을 위한 `--remote`, `--teleport` 같은 흐름을 안내합니다.

로컬로 이어받을 때는 먼저 상태를 확인합니다.

```text
웹에서 진행한 작업을 로컬에서 이어받는다.
먼저 변경 파일, 남은 TODO, 테스트 결과, 실패한 명령을 요약해줘.
아직 추가 수정은 하지 마.
```

## 체크리스트

- [ ] Web에는 잘 정의된 비동기 작업을 맡긴다.
- [ ] 범위와 비범위를 명확히 쓴다.
- [ ] 테스트 또는 완료 기준을 반드시 포함한다.
- [ ] 민감 정보가 필요한 작업은 피하거나 승인 절차를 둔다.
- [ ] 로컬로 이어받을 때 변경사항과 테스트 결과를 먼저 확인한다.
