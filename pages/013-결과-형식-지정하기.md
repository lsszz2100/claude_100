# 013. 결과 형식 지정하기

난이도: 초급  
기준일: 2026년 05월 03일
저자: AI_Innovation_Studio

## 핵심 개념

Claude에게 “답해줘”라고만 말하면 Claude는 알아서 적절한 형식을 선택합니다. 하지만 실무에서는 형식이 중요합니다. 표가 필요한지, 체크리스트가 필요한지, JSON이 필요한지, 단계별 계획이 필요한지에 따라 결과의 활용성이 달라집니다.

결과 형식을 지정하면 Claude의 답변은 더 짧고, 비교하기 쉽고, 바로 실행하기 쉬워집니다.

## 자주 쓰는 출력 형식

### 1. 표

비교, 우선순위, 위험도 정리에 좋습니다.

```text
결과를 다음 열을 가진 표로 정리해줘.
항목 | 설명 | 우선순위 | 이유 | 다음 행동
```

### 2. 체크리스트

검토, 배포, 보안 점검에 좋습니다.

```text
실행 전 확인할 체크리스트로 정리해줘.
각 항목은 체크박스 형식으로 작성해줘.
```

### 3. 단계별 계획

작업 순서가 중요할 때 사용합니다.

```text
수정하지 말고 1단계부터 5단계까지 실행 계획만 작성해줘.
각 단계에는 목적, 수정 대상, 검증 방법을 포함해줘.
```

### 4. JSON

프로그램에서 파싱해야 할 때 사용합니다.

```text
다음 JSON 형식으로만 출력해줘.
{
  "summary": "",
  "risks": [],
  "next_actions": []
}
```

### 5. 코드 리뷰 형식

리뷰 결과를 일관되게 받을 때 사용합니다.

```text
각 이슈를 다음 형식으로 작성해줘.
- 심각도:
- 파일/위치:
- 문제:
- 영향:
- 수정 제안:
- 테스트 필요 여부:
```

## 형식을 지정해야 하는 이유

형식이 없으면 Claude의 답변은 읽기 좋은 에세이가 될 수 있습니다. 하지만 실무에서는 에세이보다 실행 가능한 목록이 필요할 때가 많습니다. 특히 Claude Code에서는 계획, 변경 요약, 테스트 결과, 리뷰 결과를 구조화해야 다음 행동을 빠르게 결정할 수 있습니다.

## 실습

같은 요청을 세 가지 형식으로 받아 보세요.

```text
이 프로젝트의 개선점을 알려줘.
```

```text
이 프로젝트의 개선점을 우선순위 표로 정리해줘.
열은 개선 항목, 이유, 위험도, 예상 작업량, 검증 방법으로 해줘.
```

```text
이 프로젝트의 개선점을 체크리스트로 정리해줘.
각 항목은 30분 안에 확인 가능한 단위로 쪼개줘.
```

## Claude에게 입력할 프롬프트

```text
내 요청에 가장 적합한 출력 형식을 추천해줘.

요청:
[여기에 요청 입력]

다음 중 어떤 형식이 좋은지 판단해줘.
- 요약문
- 표
- 체크리스트
- 단계별 계획
- JSON
- 코드 리뷰 형식

추천 이유와 최종 프롬프트도 함께 작성해줘.
```

## 체크리스트

- [ ] 비교에는 표를 사용한다.
- [ ] 실행 전 점검에는 체크리스트를 사용한다.
- [ ] 복잡한 작업에는 단계별 계획을 사용한다.
- [ ] 자동화에는 JSON 또는 정해진 구조를 사용한다.
- [ ] 코드 리뷰에는 심각도와 위치를 포함한다.

## 흔한 실수

- 출력 형식을 지정하지 않는다.
- JSON을 요구하면서 설명문도 함께 요구한다.
- 너무 많은 열을 가진 표를 요구해 읽기 어렵게 만든다.
- 실행 계획과 구현을 한 번에 요구한다.
