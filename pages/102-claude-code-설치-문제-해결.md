# 102. Claude Code 설치 문제 해결

설치 문제는 대부분 Node.js, 패키지 매니저, 터미널 권한, PATH 설정, 네트워크 접근 중 하나에서 발생합니다. 오류 메시지를 바로 복사하기 전에 환경 정보를 먼저 정리하면 해결 속도가 빨라집니다.

## 먼저 확인할 것

```text
node --version
npm --version
where node
where npm
```

macOS나 Linux에서는 `where` 대신 `which`를 사용할 수 있습니다.

## 흔한 원인

| 증상 | 확인할 지점 |
| --- | --- |
| 명령을 찾을 수 없음 | PATH 등록, 터미널 재시작 |
| 설치 권한 오류 | 관리자 권한, 전역 설치 경로 |
| 패키지 다운로드 실패 | 프록시, 회사망, 방화벽, DNS |
| 실행 후 인증 실패 | 계정 로그인, 토큰, 브라우저 인증 |
| 프로젝트에서만 실패 | 현재 폴더 권한, 보안 프로그램, 긴 경로 |

## 진단 순서

설치 문제는 한 번에 고치려고 하지 말고 범위를 좁히세요.

1. 새 터미널을 열고 `node --version`을 확인한다.
2. `npm --version`으로 패키지 매니저가 실행되는지 확인한다.
3. `where node`와 `where npm`으로 실행 경로를 확인한다.
4. 회사망이나 프록시 환경이라면 다운로드 차단 여부를 확인한다.
5. Claude Code 명령이 설치된 위치와 실제 실행되는 위치가 같은지 확인한다.
6. 프로젝트 폴더가 OneDrive, 네트워크 드라이브, 권한 제한 폴더인지 확인한다.

문제가 시스템 전체인지 특정 프로젝트인지 구분하려면 빈 폴더에서 같은 명령을 실행해 보세요.

## Claude에게 줄 진단 요청

```text
Claude Code 설치가 실패했다.

환경:
- OS:
- 터미널:
- node --version:
- npm --version:
- 설치 명령:
- 오류 메시지:

요청:
1. 가능한 원인을 우선순위로 정리해줘.
2. 위험하지 않은 확인 명령부터 제안해줘.
3. 관리자 권한이나 삭제 명령이 필요한 단계는 별도 승인 단계로 분리해줘.
```

## 주의할 점

- 오류 메시지에 토큰, 이메일, 내부 URL이 있으면 가린 뒤 공유한다.
- 설치 문제를 해결하려고 임의로 시스템 폴더를 삭제하지 않는다.
- 회사 장비에서는 프록시와 보안 정책을 먼저 확인한다.
- 전역 설치와 프로젝트별 설치가 섞이면 어느 경로의 명령이 실행되는지 확인한다.
- 오래된 설치 흔적을 지우기 전에는 삭제 대상 경로를 먼저 확인한다.
