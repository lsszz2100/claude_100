# 054. 릴리스 노트 자동화

난이도: 중급  
기준일: 2026년 05월 03일
저자: AI_Innovation_Studio

## 핵심 개념

릴리스 노트는 개발자가 바꾼 파일 목록이 아니라 사용자와 운영자가 알아야 할 변경사항입니다. Claude Code는 커밋, PR, 이슈, changelog 초안을 읽고 릴리스 노트를 분류하는 데 유용합니다.

좋은 릴리스 노트는 기능, 수정, 변경, 보안, 마이그레이션, 주의사항을 구분합니다.

## 입력 자료

```text
릴리스 노트 초안을 만들기 전에 다음 자료를 확인해줘.

1. 이전 릴리스 태그
2. 이번 릴리스 범위의 커밋
3. 병합된 PR
4. 관련 이슈
5. DB migration
6. 의존성 업데이트
7. breaking change 여부
```

Git 저장소가 아니라면 변경 파일 목록이나 작업 로그를 입력 자료로 사용할 수 있습니다.

## 릴리스 노트 템플릿

```markdown
# Release Notes: [version]

## Highlights

- [사용자에게 가장 중요한 변경]

## Added

- [새 기능]

## Fixed

- [버그 수정]

## Changed

- [동작 변경]

## Security

- [보안 관련 변경]

## Migration / Deployment Notes

- [DB migration, 설정 변경, 배포 주의사항]

## Known Issues

- [알려진 문제]
```

Keep a Changelog 스타일을 쓰는 팀이라면 Added, Changed, Deprecated, Removed, Fixed, Security 분류를 맞춰도 됩니다.

## Claude Code 요청

```text
최근 변경사항을 사용자 관점 릴리스 노트로 정리해줘.

기준:
- 파일명 중심이 아니라 사용자 영향 중심으로 작성
- 기능, 수정, 변경, 보안, 마이그레이션, 알려진 문제로 분류
- breaking change와 운영 주의사항은 따로 표시
- 근거가 없는 내용은 쓰지 않기

아직 CHANGELOG 파일은 수정하지 마.
```

## 개발자용과 사용자용 분리

릴리스 노트에는 두 관점이 있습니다.

```text
같은 변경사항을 두 버전으로 작성해줘.

1. 사용자용 릴리스 노트
- 쉬운 표현
- 기능과 영향 중심

2. 개발자/운영자용 릴리스 노트
- migration, 환경 변수, API 변경, 배포 순서 포함
```

## 위험한 표현 줄이기

나쁜 예:

```text
- 여러 버그 수정
- 성능 개선
- 내부 코드 정리
```

좋은 예:

```text
- 고객 목록에서 검색어를 빠르게 바꿀 때 이전 결과가 남던 문제를 수정했다.
- 주문 목록 API에 pagination을 적용해 대량 데이터 조회 시간을 줄였다.
- 결제 설정에 `PAYMENT_WEBHOOK_SECRET` 환경 변수가 추가되었다.
```

구체적이어야 사용자가 행동할 수 있습니다.

## 실습

최근 변경사항 3개를 골라 사용자용 릴리스 노트와 운영자용 릴리스 노트로 각각 바꿔 보세요.

## Claude Code에 입력할 프롬프트

```text
이번 릴리스 노트 초안을 작성해줘.

입력 범위:
[태그/커밋 범위/PR 목록/변경 파일]

출력:
1. 사용자용 요약
2. Added
3. Fixed
4. Changed
5. Security
6. Migration / Deployment Notes
7. Known Issues

제약:
- 근거 없는 내용을 쓰지 마.
- 내부 구현만 바뀐 내용은 사용자 영향이 있을 때만 포함해줘.
- breaking change와 운영 주의사항은 강조해줘.
- 아직 파일은 수정하지 마.
```

## 체크리스트

- [ ] 사용자 영향 중심으로 쓴다.
- [ ] 기능, 수정, 변경, 보안, 마이그레이션을 구분한다.
- [ ] breaking change를 숨기지 않는다.
- [ ] 운영자가 해야 할 일을 명확히 적는다.
- [ ] 근거 없는 홍보 문구를 넣지 않는다.

## 흔한 실수

- 커밋 메시지를 그대로 붙인다.
- “성능 개선”처럼 측정 없는 표현을 쓴다.
- 마이그레이션이나 환경 변수 변경을 빠뜨린다.
- 알려진 문제를 숨긴다.
