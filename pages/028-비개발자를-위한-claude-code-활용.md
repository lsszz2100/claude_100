# 028. 비개발자를 위한 Claude Code 활용

난이도: 초급  
기준일: 2026년 05월 03일

## 핵심 개념

Claude Code는 개발자만 쓰는 도구가 아닙니다. 폴더 안의 파일을 읽고, 정리하고, 비교하고, 문서 초안을 만들고, 반복 작업을 자동화하는 데도 사용할 수 있습니다.

비개발자에게 중요한 것은 코드를 잘 아는 것이 아니라 “파일과 작업 기준을 명확히 설명하는 것”입니다. Claude Code는 사용자가 정한 범위 안에서 파일을 다루는 작업 파트너가 될 수 있습니다.

## 비개발자가 맡길 수 있는 작업

| 작업 | 예시 |
| --- | --- |
| 문서 요약 | 회의록 20개에서 결정사항만 추출 |
| 파일 정리 | 폴더별 파일명 규칙 제안 |
| 체크리스트 작성 | 배포 전 확인 항목 정리 |
| 표 변환 | Markdown 표를 CSV 구조로 변환 |
| 비교 | 두 문서의 차이점과 충돌 내용 정리 |
| 초안 작성 | 고객 안내문, FAQ, 운영 매뉴얼 초안 |
| 간단한 자동화 | 반복 파일명 변경 스크립트 설계 |

단, 민감 정보가 있는 폴더는 주의해야 합니다. 고객 개인정보, 비밀번호, 계약서, 내부 재무 자료가 섞여 있다면 먼저 읽기 범위를 좁히세요.

## 안전한 첫 요청

처음에는 읽기 전용으로 시작하세요.

```text
현재 폴더의 파일 구조를 확인해줘.
아직 파일은 수정하거나 이동하지 마.

출력:
1. 폴더별 파일 종류
2. 중복되거나 정리할 만한 파일 후보
3. 민감 정보가 있을 수 있어 주의해야 할 파일명
4. 정리 작업을 시작하기 전 확인 질문
```

이 요청은 실제 파일을 바꾸지 않고 상황을 파악하게 합니다.

## 문서 정리 요청 예시

회의록 폴더를 정리하고 싶다면 이렇게 요청할 수 있습니다.

```text
meetings 폴더의 Markdown 파일을 읽고 회의별 핵심 내용을 요약해줘.

제약:
- 파일은 수정하지 마.
- 개인정보처럼 보이는 내용은 그대로 인용하지 마.
- 결정사항, 담당자, 마감일, 미해결 질문만 추출해줘.

출력:
- 파일명
- 회의 주제
- 결정사항
- 액션 아이템
- 확인 필요
```

## 파일 이름 정리 요청 예시

파일명을 바꾸는 작업은 되돌리기 어려울 수 있으므로 먼저 계획만 받아야 합니다.

```text
현재 폴더의 파일명 규칙을 분석해줘.
아직 파일명은 바꾸지 마.

요청:
1. 현재 파일명 패턴을 분류해줘.
2. 추천 파일명 규칙을 제안해줘.
3. 바꾸면 위험한 파일을 알려줘.
4. 변경 전/후 매핑표를 만들어줘.
5. 내가 승인하기 전에는 어떤 파일도 이동하거나 이름을 바꾸지 마.
```

## 자동화 요청 예시

비개발자도 작은 자동화 도구를 만들 수 있습니다. 다만 처음부터 실행하지 말고 설명과 계획을 먼저 받으세요.

```text
매주 반복하는 보고서 정리 작업을 자동화하고 싶다.

현재 작업:
- downloads 폴더에서 이번 주 CSV 파일을 찾는다.
- 파일명을 날짜_고객사_리포트.csv 형식으로 바꾼다.
- reports 폴더로 옮긴다.
- 처리한 파일 목록을 기록한다.

요청:
- 자동화 가능 여부를 판단해줘.
- 필요한 입력과 위험을 정리해줘.
- 실행 전에 검토할 계획만 작성해줘.
```

## 비개발자가 특히 조심할 점

Claude Code는 파일을 실제로 수정할 수 있습니다. 그래서 다음 문장을 자주 쓰세요.

```text
아직 파일은 수정하지 마.
먼저 계획과 변경 예상 목록만 보여줘.
```

파일 이동, 삭제, 이름 변경, 일괄 변환은 항상 변경 전/후 목록을 먼저 받아야 합니다.

## 실습

문서가 들어 있는 작은 폴더를 하나 고르고 다음을 해 보세요.

1. 폴더 구조만 요약하게 한다.
2. 파일을 수정하지 않고 정리 후보를 찾게 한다.
3. 파일명 규칙을 제안하게 한다.
4. 변경 전/후 매핑표를 만들게 한다.
5. 민감 정보가 있을 수 있는 파일을 표시하게 한다.

## Claude Code에 입력할 프롬프트

```text
나는 개발자가 아니지만 이 폴더를 정리하고 싶다.

목표:
- 문서와 자료 파일을 이해하기 쉽게 정리한다.

제약:
- 아직 파일을 수정, 삭제, 이동, 이름 변경하지 마.
- 먼저 현재 구조를 분석하고 계획만 제안해줘.
- 개인정보나 비밀정보처럼 보이는 내용은 그대로 출력하지 마.

출력:
1. 현재 폴더 구조 요약
2. 정리 후보
3. 위험한 작업
4. 추천 파일명 규칙
5. 변경 전/후 예상표
6. 내가 승인하기 전 확인해야 할 질문
```

## 체크리스트

- [ ] 처음 요청은 읽기 전용으로 시작한다.
- [ ] 파일 수정, 이동, 삭제 전 계획을 먼저 받는다.
- [ ] 민감 정보가 있는지 확인한다.
- [ ] 변경 전/후 매핑표를 만든다.
- [ ] 자동화는 작은 샘플로 먼저 검증한다.

## 흔한 실수

- “폴더 정리해줘”라고 바로 실행을 맡긴다.
- 고객 정보나 계약서를 범위 제한 없이 읽히게 한다.
- 파일명 변경 목록을 확인하지 않고 승인한다.
- 자동화 스크립트를 샘플 테스트 없이 전체 폴더에 적용한다.
