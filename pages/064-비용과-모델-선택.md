# 064. 비용과 모델 선택

난이도: 중급  
기준일: 2026년 05월 03일

## 핵심 개념

Claude Code 비용은 질문 횟수보다 컨텍스트 크기, 읽은 파일 수, 대화 길이, 작업 복잡도, compact 빈도에 더 크게 영향을 받을 수 있습니다. 공식 문서에서도 `/cost`, `/compact`, `/clear`, 구체적 요청, 작업 분할을 비용 관리 방법으로 안내합니다.

모델 선택은 “항상 가장 강한 모델”이 아니라 작업 위험도와 복잡도에 맞추는 문제입니다.

## 비용 확인

현재 세션 사용량은 `/cost`로 확인합니다.

```text
/cost
```

단, 구독 유형에 따라 `/cost`의 의미와 표시 방식은 다를 수 있습니다.

## 비용을 늘리는 패턴

- “전체 프로젝트 다 읽고” 같은 넓은 요청
- 대용량 로그 전체 붙여넣기
- 긴 세션에서 주제 전환을 반복하기
- 같은 파일을 여러 번 불필요하게 다시 읽기
- 실패한 작업을 원인 분석 없이 반복하기
- 여러 에이전트/자동화를 동시에 과하게 실행하기

## 비용을 줄이는 요청

```text
이번 작업은 로그인 실패 원인 조사다.
먼저 auth 관련 파일과 실패 로그의 최초 오류만 확인해줘.
전체 프로젝트 요약은 하지 마.
```

```text
이 문제를 3단계로 나눠줘.
먼저 1단계 조사만 진행하고, 구현은 승인 후 진행해줘.
```

구체적인 범위 제한이 비용 관리의 핵심입니다.

## 모델 선택 기준

| 작업 | 권장 방향 |
| --- | --- |
| 단순 문서 요약 | 빠르고 저렴한 모델 또는 낮은 reasoning |
| 오타/작은 문서 수정 | 낮은 effort |
| 버그 원인 조사 | 중간 이상 |
| 보안/인증/DB 변경 | 강한 모델과 계획 우선 |
| 대규모 리팩터링 | 강한 모델, 작은 단계, 테스트 필수 |
| 아키텍처 설계 | 강한 모델, 충분한 reasoning |

Claude Code에서는 `/model`로 모델을 확인하거나 바꿀 수 있습니다.

```text
/model
```

## 작업 분할 프롬프트

```text
이 작업을 비용과 위험을 줄이도록 단계별로 나눠줘.

조건:
- 1단계는 읽기 전용 조사
- 2단계는 최소 수정
- 3단계는 테스트와 검토
- 각 단계마다 읽을 파일 범위를 제한
- 다음 단계로 넘어가기 전 승인 필요
```

## 체크리스트

- [ ] `/cost`로 사용량을 확인한다.
- [ ] 넓은 코드베이스 탐색을 피한다.
- [ ] 작업을 조사, 구현, 검증으로 나눈다.
- [ ] 주제가 바뀌면 `/clear`를 쓴다.
- [ ] 위험한 작업에는 더 강한 모델과 계획 단계를 쓴다.
