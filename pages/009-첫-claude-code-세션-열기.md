# 009. 첫 Claude Code 세션 열기

난이도: 초급  
기준일: 2026년 05월 03일
저자: AI_Innovation_Studio

## 핵심 개념

첫 Claude Code 세션에서 가장 중요한 것은 “수정”이 아니라 “이해”입니다. Claude Code를 처음 열자마자 기능을 만들거나 버그를 고치게 하면 프로젝트 구조를 충분히 파악하지 못한 상태에서 작업이 시작될 수 있습니다. 처음에는 프로젝트를 읽고 요약하게 하세요.

## 첫 세션의 목표

첫 세션에서는 다음을 알아내는 것이 좋습니다.

- 이 프로젝트가 무엇을 하는가
- 어떤 언어와 프레임워크를 쓰는가
- 실행 명령은 무엇인가
- 테스트 명령은 무엇인가
- 핵심 폴더는 어디인가
- 위험한 파일이나 민감한 설정은 무엇인가
- 다음에 할 만한 작은 작업은 무엇인가

## 추천 첫 요청

```text
이 프로젝트를 처음 보는 개발자에게 설명하듯 분석해줘.
아직 파일은 수정하지 마.

다음 항목을 알려줘.
1. 프로젝트 목적
2. 기술 스택
3. 폴더 구조
4. 실행 방법
5. 테스트 방법
6. 핵심 파일 5개
7. 주의해야 할 설정 파일
8. 다음에 하기 좋은 작은 작업 5개
```

이 요청은 안전합니다. Claude Code가 파일을 읽을 수는 있지만, 수정하지 않도록 명확히 지시했기 때문입니다.

## 세션 중 확인할 것

Claude Code가 어떤 파일을 읽는지 주의 깊게 보세요. README만 읽고 결론을 내리는지, 설정 파일과 소스 파일까지 함께 보는지 확인해야 합니다. 좋은 분석은 실제 파일에 근거합니다. 추측이 많다면 “실제 파일을 근거로 다시 확인해줘”라고 요청하세요.

## 첫 수정은 작게

첫 수정은 작고 되돌리기 쉬운 작업이어야 합니다.

좋은 첫 수정 예:

- README의 실행 명령 보완
- 오타 수정
- 작은 테스트 하나 추가
- 명확한 에러 메시지 개선
- 사용하지 않는 import 제거

나쁜 첫 수정 예:

- 전체 아키텍처 변경
- 인증 시스템 교체
- 데이터베이스 대규모 마이그레이션
- 모든 파일 포맷 변경
- 테스트 없이 대규모 리팩터링

## Claude Code에 입력할 프롬프트

```text
첫 번째 수정 작업을 추천해줘.
조건:
- 30분 안에 끝낼 수 있어야 한다.
- 파일 변경은 1~3개로 제한한다.
- 테스트 또는 검증 방법이 있어야 한다.
- 되돌리기 쉬워야 한다.

추천 작업 5개를 난이도순으로 제안해줘.
```

## 체크리스트

- [ ] 첫 세션에서 파일 수정을 바로 요구하지 않는다.
- [ ] 프로젝트 목적과 실행 방법을 먼저 확인한다.
- [ ] 첫 수정은 작고 검증 가능해야 한다.
- [ ] Claude의 분석이 실제 파일에 근거하는지 확인한다.
