이 책의 목표는 Claude를 잘 묻는 도구로만 사용하는 수준을 넘어, Claude Code와 함께 실제 프로젝트를 읽고, 수정하고, 테스트하고, 배포하고, 운영하는 능력을 갖추는 것입니다.

2026년 05월 03일 기준으로 Claude 생태계는 웹 대화, 데스크톱 앱, 터미널 기반 Claude Code, IDE 통합, MCP, Skills, Hooks, Subagents, Agent Teams, Agent SDK까지 확장되어 있습니다. 이 책은 그 요소들을 흩어진 기능으로 설명하지 않고 하나의 실전 흐름으로 묶습니다.

처음 AI 도구를 배우는 분은 기능 이름이 많아 보일 수 있습니다. 하지만 실제 사용 순서는 단순합니다. 먼저 좋은 질문을 만들고, 그다음 프로젝트 파일을 읽게 하고, 작은 변경을 맡기고, 테스트와 리뷰로 확인합니다. 이 기본 흐름이 익숙해진 뒤에 Skills, Hooks, MCP, Subagents 같은 확장 기능을 붙이면 됩니다.

이 책은 Claude가 모든 판단을 대신해 준다고 말하지 않습니다. 오히려 반대입니다. Claude를 잘 쓰려면 사용자가 목표, 범위, 금지사항, 검증 방법을 더 분명하게 써야 합니다. 그 과정에서 초보자는 작업을 구조화하는 법을 배우고, 실무자는 반복 업무를 줄이며, 팀은 공통된 작업 기준을 만들 수 있습니다.

초급자는 001번부터 순서대로 읽으세요. 개발 경험이 있다면 031번부터 시작해도 됩니다. Claude Code를 이미 사용 중이라면 066번 Skills, 075번 Hooks, 083번 MCP, 092번 Subagents부터 읽고 필요한 부분으로 돌아오면 됩니다.

각 페이지는 개념, 이유, 실습, 프롬프트, 체크리스트 순서로 읽으면 됩니다. 개념을 읽고 바로 이해되지 않아도 괜찮습니다. 실습 프롬프트를 한 번 실행해 보면 어떤 맥락을 더 줘야 하는지 감이 잡힙니다.

중요한 것은 한 번에 큰 자동화를 만들지 않는 것입니다. 처음에는 문서 요약, 폴더 구조 설명, 테스트 실패 분석처럼 되돌리기 쉬운 작업부터 시작하세요. 파일 수정, 배포, 외부 서비스 연결은 충분히 익숙해진 뒤에 다루는 편이 안전합니다.
