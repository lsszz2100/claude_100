# 029. 학습자용 Claude 사용법

난이도: 초급  
기준일: 2026년 05월 03일

## 핵심 개념

Claude는 학습에 매우 유용하지만, 잘못 쓰면 생각하는 과정을 건너뛰게 만듭니다. 학습자는 Claude에게 정답을 바로 받기보다 설명, 힌트, 반례, 연습문제, 피드백을 요청하는 방식으로 써야 합니다.

학습용 Claude 사용의 핵심은 “나 대신 풀어줘”가 아니라 “내가 풀 수 있게 도와줘”입니다.

## 좋은 학습 요청

학습 요청에는 현재 수준과 원하는 도움 방식을 포함하세요.

```text
나는 JavaScript 초급자다.
Promise와 async/await 차이를 배우고 있다.

요청:
- 먼저 비유 없이 정확한 개념으로 설명해줘.
- 그 다음 짧은 예제를 보여줘.
- 마지막에 내가 풀 연습문제 3개를 내줘.
- 정답은 바로 보여주지 말고, 내가 답한 뒤 피드백해줘.
```

이 요청은 Claude가 강의, 예제, 연습, 피드백 순서로 도와주게 만듭니다.

## 정답을 숨기는 요청

문제를 풀 때는 Claude에게 정답을 바로 말하지 말라고 요청할 수 있습니다.

```text
아래 문제를 풀고 싶다.
정답을 바로 말하지 마.

대신:
1. 문제를 이해하는 데 필요한 개념을 설명해줘.
2. 첫 번째 힌트만 줘.
3. 내가 답을 제출하면 틀린 부분만 짚어줘.
4. 그래도 못 풀면 다음 힌트를 줘.
```

이 방식은 코딩 테스트, 수학, 글쓰기, 영어 학습 모두에 쓸 수 있습니다.

## 코드 학습에 쓰는 법

코드를 배울 때는 “이 코드 설명해줘”보다 구체적인 질문이 좋습니다.

```text
아래 코드를 초급자가 이해할 수 있게 설명해줘.

출력 형식:
1. 이 코드가 하는 일 한 문장
2. 중요한 변수와 함수
3. 실행 순서
4. 헷갈리기 쉬운 부분
5. 내가 바꿔볼 만한 실험 3개

마지막에는 비슷한 코드를 직접 작성하는 연습문제를 내줘.
```

단순 설명보다 “직접 바꿔볼 실험”을 요청하면 학습 효과가 커집니다.

## 틀린 답 피드백 받기

Claude에게 자신의 답을 검토하게 할 때는 정답만 요구하지 말고 사고 과정을 봐 달라고 하세요.

```text
내 풀이를 검토해줘.

요청:
- 최종 답만 보지 말고 풀이 과정에서 잘못된 가정을 찾아줘.
- 맞은 부분과 틀린 부분을 구분해줘.
- 바로 정답을 다시 쓰지 말고, 내가 고칠 수 있는 힌트를 먼저 줘.
```

이 요청은 학습자가 어디서 틀렸는지 이해하게 합니다.

## 설명 수준 조절

Claude의 설명이 너무 쉽거나 어렵다면 수준을 조절하세요.

```text
방금 설명은 너무 추상적이다.
중학생도 이해할 수 있는 수준으로 다시 설명하되,
정확한 용어는 유지해줘.
```

또는:

```text
기본 설명은 이해했다.
이제 실무 개발자가 알아야 할 예외 상황과 흔한 버그를 중심으로 설명해줘.
```

좋은 학습은 같은 주제를 여러 수준으로 오가며 이해하는 과정입니다.

## Claude Code로 학습하기

Claude Code는 실제 프로젝트를 읽으면서 학습할 때 유용합니다.

```text
이 프로젝트에서 로그인 흐름을 배우고 싶다.

제약:
- 파일은 수정하지 마.
- 관련 파일을 찾아서 읽고, 초급자용 학습 순서로 설명해줘.
- 먼저 봐야 할 파일, 각 파일의 역할, 따라가야 할 함수 흐름을 알려줘.
- 마지막에 내가 이해했는지 확인할 질문 5개를 내줘.
```

이렇게 하면 코드베이스가 학습 교재가 됩니다.

## 실습

지금 배우는 주제 하나를 고르고 다음 방식으로 Claude에게 요청해 보세요.

1. 현재 수준을 알려준다.
2. 정답을 바로 말하지 말라고 한다.
3. 힌트를 단계별로 요청한다.
4. 자신의 답을 제출하고 피드백을 받는다.
5. 비슷한 문제를 새로 만들어 달라고 한다.

## Claude에 입력할 프롬프트

```text
나는 [주제]를 배우고 있는 [초급/중급] 학습자다.

학습 목표:
- [무엇을 이해하고 싶은지 작성]

도움 방식:
- 정답을 바로 말하지 마.
- 먼저 필요한 개념을 설명해줘.
- 그 다음 힌트 1개만 줘.
- 내가 답을 쓰면 풀이 과정의 오류를 피드백해줘.
- 마지막에는 비슷한 연습문제 3개를 내줘.

설명 수준:
- 전문 용어는 쓰되, 처음 나올 때는 쉽게 풀어 설명해줘.
```

## 체크리스트

- [ ] 현재 수준을 Claude에게 알려준다.
- [ ] 정답을 바로 말하지 말라고 요청한다.
- [ ] 힌트, 피드백, 연습문제를 활용한다.
- [ ] 설명이 어렵거나 쉬우면 수준을 조절한다.
- [ ] 실제 프로젝트에서는 읽기 전용으로 학습을 시작한다.

## 흔한 실수

- 숙제나 문제의 정답만 복사한다.
- 이해하지 못한 개념을 넘어간다.
- Claude의 설명을 검증 없이 외운다.
- 직접 풀어보는 시간을 생략한다.
