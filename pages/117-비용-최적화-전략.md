# 117. 비용 최적화 전략

AI 비용 최적화는 무조건 저렴한 모델을 쓰는 일이 아닙니다. 작업의 난이도, 실패 비용, 지연 요구사항, 검증 방식에 맞춰 모델과 컨텍스트를 배분하는 일입니다.

## 비용이 늘어나는 원인

- 너무 긴 컨텍스트를 매번 보낸다.
- 필요 없는 파일까지 읽는다.
- 같은 질문을 반복 처리한다.
- 실패 시 무제한 재시도한다.
- 고난도 모델을 단순 분류 작업에 쓴다.
- 평가 없이 프롬프트를 계속 늘린다.

## 최적화 방법

| 방법 | 설명 |
| --- | --- |
| 컨텍스트 축소 | 필요한 파일과 요약만 전달 |
| 캐싱 | 반복되는 문서 요약과 검색 결과 재사용 |
| 모델 라우팅 | 쉬운 작업과 어려운 작업 분리 |
| 배치 처리 | 독립 작업을 묶어 처리 |
| 평가셋 | 품질 저하 없이 줄였는지 확인 |
| 실패 제한 | 재시도 횟수와 fallback 지정 |

## Claude에게 줄 요청

```text
다음 AI 워크플로의 비용 최적화 방안을 제안해줘.

워크플로:
[설명]

현재 문제:
- 비용:
- 지연:
- 품질:

요청:
1. 비용이 많이 드는 지점
2. 줄여도 되는 컨텍스트
3. 캐싱 후보
4. 모델 라우팅 후보
5. 품질 저하를 확인할 평가 방법
```

## 체크리스트

- [ ] 비용을 줄이기 전 기준 품질을 측정했다.
- [ ] 컨텍스트를 줄인 뒤 품질을 다시 평가했다.
- [ ] 실패 비용이 큰 작업은 무리하게 낮은 모델로 보내지 않는다.
- [ ] 비용 지표를 운영 대시보드에서 볼 수 있다.
