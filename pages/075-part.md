# Part 08. Hooks

이 장은 고급 단계 독자를 위한 흐름입니다. 각 페이지는 하나의 주제를 다루며, 개념 설명과 실습 프롬프트를 함께 제공합니다.

## 처음 읽는 분께

이 파트는 용어를 외우는 장이 아니라 실제 작업에서 판단 기준을 세우는 장입니다. 각 항목을 읽을 때는 “언제 쓰는가”, “무엇을 조심해야 하는가”, “어떻게 검증하는가”를 함께 확인하세요.

초보자는 본문을 읽은 뒤 바로 실습 프롬프트를 실행해 보는 것이 좋습니다. 실행 결과가 예상과 다르면 실패로 보지 말고, 어떤 맥락이 부족했는지 기록하세요. 그 기록이 다음 요청을 더 정확하게 만드는 재료가 됩니다.

## 이 장에서 다루는 100선 항목

- 075. Hooks 개념: Hooks는 Claude Code 이벤트에 반응해 자동으로 실행되는 검증/자동화 장치다.
- 076. PreToolUse Hook: 도구 실행 전 차단, 확인, 로깅을 수행한다.
- 077. PostToolUse Hook: 도구 실행 후 포맷, 테스트, 스캔, 알림을 수행한다.
- 078. 파일 수정 전 보안 차단: .env, secret, production config 같은 민감 파일 보호 전략을 세운다.
- 079. 파일 수정 후 자동 포맷: 자동 포맷은 리뷰 비용을 줄이지만 실패 시 명확한 fallback이 필요하다.
- 080. 커밋 전 테스트 Hook: 커밋 전 최소 테스트를 자동 실행해 회귀를 줄인다.
- 081. 명령어 로깅 Hook: Claude가 실행한 명령을 기록하면 감사와 디버깅이 쉬워진다.
- 082. Hooks 보안 설계: Hook 자체가 위험한 자동 실행 지점이 될 수 있으므로 최소 권한으로 설계한다.

## 학습 방식

각 항목을 읽은 뒤 바로 Claude 또는 Claude Code에 실습 프롬프트를 입력해 보세요. 읽기만 하면 기능 목록으로 남고, 직접 실행하면 작업 습관으로 바뀝니다.
