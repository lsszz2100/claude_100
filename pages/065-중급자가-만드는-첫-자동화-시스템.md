# 065. 중급자가 만드는 첫 자동화 시스템

난이도: 중급  
기준일: 2026년 05월 03일
저자: AI_Innovation_Studio

## 핵심 개념

중급자의 첫 자동화 시스템은 거창할 필요가 없습니다. 매일 반복하는 개발 상태 요약, 실패한 테스트 정리, PR 본문 생성, 릴리스 노트 초안처럼 입력과 출력이 분명한 작업부터 시작하면 됩니다.

목표는 전면 자동화가 아니라 “반복 판단을 줄이고, 검증 가능한 초안을 만드는 것”입니다.

## 좋은 첫 자동화 후보

| 자동화 | 입력 | 출력 |
| --- | --- | --- |
| 일일 개발 상태 요약 | git status, diff, 테스트 결과 | 오늘 할 일 요약 |
| PR 본문 생성 | diff, 커밋, 테스트 결과 | PR Markdown |
| 실패 테스트 요약 | test log | 최초 원인과 수정 후보 |
| 릴리스 노트 초안 | merged PR 목록 | 사용자용 변경 요약 |
| README 검증 | README, package 설정 | 오래된 명령 목록 |

## 첫 자동화 설계

```markdown
# Automation: Daily Dev Summary

## Goal

- 매일 작업 시작 전에 현재 저장소 상태를 요약한다.

## Inputs

- git status
- git diff --stat
- 최근 테스트 로그
- progress.md

## Outputs

- 현재 변경 파일
- 위험한 변경
- 마지막 테스트 결과
- 오늘 먼저 할 작업 3개

## Safety

- 파일 수정 없음
- 명령은 읽기 전용만 사용
- 비밀정보 출력 금지
```

## Claude Code용 프롬프트

```text
매일 아침 실행할 개발 상태 요약 자동화를 설계해줘.

요구사항:
- 읽기 전용이어야 한다.
- git status와 diff 요약을 포함한다.
- progress.md가 있으면 함께 읽는다.
- 마지막 테스트 결과가 있으면 요약한다.
- 오늘 먼저 할 작업 3개를 제안한다.

제약:
- 아직 스크립트는 만들지 마.
- 필요한 명령과 출력 형식만 설계해줘.
- 비밀정보는 출력하지 않는 규칙을 포함해줘.
```

## slash command로 만들기

반복 프롬프트는 `.claude/commands/` 아래 custom slash command로 만들 수 있습니다.

예시 파일:

```text
.claude/commands/dev-summary.md
```

내용 예시:

```markdown
---
description: Summarize current development status
allowed-tools: Bash(git status:*), Bash(git diff:*), Bash(git log:*)
---

Summarize the current development status.

Use:
- git status
- git diff --stat
- git log --oneline -5

Output:
1. Current branch
2. Changed files
3. Risky changes
4. Suggested next steps

Do not modify files.
Do not print secrets.
```

사용:

```text
/dev-summary
```

## 첫 버전 원칙

- 읽기 전용으로 시작한다.
- destructive 명령을 넣지 않는다.
- 출력 형식을 고정한다.
- 성공 기준을 명확히 한다.
- 사용해 보며 조금씩 개선한다.

## 실습

자신의 프로젝트에 `dev-summary` 명령 초안을 설계해 보세요. 처음에는 파일을 만들지 말고, Claude Code에게 명령 내용만 제안하게 합니다.

## Claude Code에 입력할 프롬프트

```text
내 프로젝트에 맞는 첫 custom slash command를 설계해줘.

목표:
- 매일 개발 상태를 읽기 전용으로 요약한다.

제약:
- destructive 명령 금지
- 파일 수정 금지
- allowed-tools는 필요한 git 읽기 명령만 허용
- 출력 형식은 고정

출력:
`.claude/commands/dev-summary.md`에 넣을 Markdown 초안
```
