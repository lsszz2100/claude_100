# 113. JSON 출력 검증 패턴

AI가 JSON을 출력해야 하는 작업에서는 “JSON처럼 보이는 텍스트”와 “프로그램이 파싱할 수 있는 JSON”을 구분해야 합니다. 운영 코드에서는 반드시 파서와 스키마로 검증하세요.

## 기본 요청 패턴

```text
아래 스키마를 만족하는 JSON만 출력해줘.
설명 문장, Markdown 코드블록, 주석은 출력하지 마.

Schema:
{
  "type": "object",
  "required": ["title", "items"],
  "properties": {
    "title": { "type": "string" },
    "items": {
      "type": "array",
      "items": { "type": "string" }
    }
  }
}
```

## 검증 단계

1. JSON 파싱이 되는지 확인한다.
2. 필수 필드가 있는지 확인한다.
3. 타입이 맞는지 확인한다.
4. 값의 길이와 허용 범위를 확인한다.
5. 실패 시 재요청 또는 fallback을 실행한다.

## 스키마 설계 팁

- 문자열 길이의 최소값과 최대값을 정한다.
- enum으로 허용 값을 제한한다.
- 배열의 최대 개수를 정한다.
- 숫자는 최소값과 최대값을 정한다.
- 알 수 없는 필드는 거부한다.
- 사용자에게 보여줄 문장과 내부 처리용 값을 분리한다.

예를 들어 상태값을 자유 문자열로 받으면 `done`, `completed`, `finish`처럼 표현이 섞입니다. 운영 코드에서는 enum으로 `pending`, `running`, `failed`, `succeeded`처럼 허용 값을 좁히는 편이 안전합니다.

## 실패 대응 예시

```text
이전 출력은 JSON 파싱에 실패했다.
오류:
[파서 오류]

다시 출력해줘.
조건:
- JSON 객체 하나만 출력
- Markdown 코드블록 금지
- 주석 금지
- 스키마에 없는 필드 금지
```

## Claude에게 줄 요청

```text
다음 AI 출력 요구사항을 JSON 스키마와 검증 절차로 바꿔줘.

요구사항:
[설명]

포함할 것:
1. JSON Schema 초안
2. 정상 예시
3. 실패 예시
4. 재요청 프롬프트
5. 운영 코드에서 검증할 항목
```

## 체크리스트

- [ ] 코드블록 없이 JSON만 받도록 요청했다.
- [ ] 파서와 스키마 검증을 모두 수행한다.
- [ ] 실패 시 재시도 횟수와 fallback이 있다.
- [ ] 모델 출력을 신뢰하고 바로 DB에 저장하지 않는다.
- [ ] 스키마 밖의 필드를 허용할지 명시했다.
- [ ] 실패한 출력 샘플을 평가셋에 추가한다.
