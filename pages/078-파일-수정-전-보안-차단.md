# 078. 파일 수정 전 보안 차단

난이도: 고급  
기준일: 2026년 05월 03일

## 핵심 개념

파일 수정 전 보안 차단은 `PreToolUse` Hook의 대표적인 사용 사례입니다. Claude가 `Edit`, `Write`, `MultiEdit`를 실행하기 전에 민감 파일, 운영 설정, 생성 파일을 막을 수 있습니다.

## 차단 후보

```text
.env
.env.*
secrets/
credentials.json
*.pem
*.key
infra/prod/
k8s/prod/
terraform/prod/
src/generated/
```

모든 차단 후보가 모든 프로젝트에 맞지는 않습니다. 팀의 실제 위험 파일 기준으로 정해야 합니다.

## 설정 예시

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit",
        "hooks": [
          {
            "type": "command",
            "command": "python .claude/hooks/block_sensitive_edits.py"
          }
        ]
      },
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "python .claude/hooks/block_sensitive_edits.py"
          }
        ]
      }
    ]
  }
}
```

## 스크립트 예시

```python
import fnmatch
import json
import sys

data = json.load(sys.stdin)
path = data.get("tool_input", {}).get("file_path", "")

blocked_patterns = [
    ".env",
    ".env.*",
    "secrets/*",
    "*.pem",
    "*.key",
    "infra/prod/*",
    "k8s/prod/*",
    "src/generated/*",
]

for pattern in blocked_patterns:
    if fnmatch.fnmatch(path, pattern) or fnmatch.fnmatch(path.replace("\\", "/"), pattern):
        print(
            f"Blocked edit to sensitive or generated path: {path}. Ask the user for explicit approval or edit the source generator instead.",
            file=sys.stderr,
        )
        sys.exit(2)

sys.exit(0)
```

## Hook와 permissions.deny의 차이

`.claude/settings.json`의 `permissions.deny`는 도구 권한 차단에 적합합니다. Hook은 더 세밀한 로직과 안내 메시지가 필요할 때 유용합니다.

두 방식을 함께 쓸 수 있습니다.

- `permissions.deny`: 명확한 읽기/쓰기 차단
- `PreToolUse Hook`: 조건부 차단, 상세 메시지, 팀 정책 검사

## 예외 설계

차단 규칙에는 예외도 필요합니다. 예를 들어 `.env`는 차단해야 하지만 `.env.example`은 문서와 온보딩에 필요할 수 있습니다. 생성 파일도 직접 수정은 막되, 생성 원본이나 스키마 수정은 허용해야 할 수 있습니다.

예외를 설계할 때는 다음 질문을 사용하세요.

- 이 파일을 수정해야 하는 합법적인 상황이 있는가?
- 수정이 필요하다면 어떤 승인 절차가 필요한가?
- 대체로 수정해야 할 원본 파일이 따로 있는가?
- 변경 후 어떤 검증을 해야 하는가?

## 차단 메시지 예시

```text
Blocked edit to production config.
Reason: production 설정은 직접 수정할 수 없습니다.
Next: staging 설정에서 변경을 검증한 뒤 승인 절차를 진행하세요.
```
